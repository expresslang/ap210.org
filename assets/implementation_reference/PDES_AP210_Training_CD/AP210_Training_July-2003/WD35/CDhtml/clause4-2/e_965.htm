4.2.965 Work_request

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Work_request is a type of Managed_design_object. A Work_request is either a Change_request or a Start_request, a document that identifies the work to be done to initiate or redefine the design of a part.

The data associated with this application object are the following:

  • description
  • implementation_order
  • initiator
  • reason
  • referenced_product
  • request_date
  • request_recipient
  • status
  • work_request_approval
  • work_request_id

4.2.965.1 description

The description specifies the Ee_text information of the work to be done to create or change the design of a product by a Work_request.

4.2.965.2 implementation_order

The implementation_order is an inverse relationship that specifies the Work_order that specifies the Work_request as its referenced_work_request. There shall be no more than one Work_order for a Work_request.

4.2.965.3 initiator

The initiator specifies either the Person_and_organization or the Organization that initiates the Work_request. There shall be one or more Person_and_organization or Organization for a Work_request.

4.2.965.4 reason

The reason specifies the Ee_text or the Evaluated_characteristic which is the purpose for generating the Work_request.

4.2.965.5 referenced_product

The referenced_product specifies the specific Ee_product_version that shall be the subject of the Work_request. There shall be one or more Ee_product_version for the Work_request.

4.2.965.6 request_date

The request_date specifies either the Date or the Date_and_time when the Work_request was initiated.

4.2.965.7 request_recipient

The request_recipient specifies either the Person_and_organization or the Organization that will receive the Work_request and take whatever action is called out in it. There shall be one or more Person_and_organization or Organization for a Work_request.

4.2.965.8 status

The status specifies the text that describes the current level of completion of the Work_request.

EXAMPLE: Example values for the status of a Work_request may be proposed, in-work, issued, and hold.

4.2.965.9 work_request_approval

The work_request_approval specifies the Ee_approval for the Work_request before it is to be acted upon.

4.2.965.10 work_request_id

The work_request_id specifies the Ee_text identifier for the proposed work on a product's design.

Formal constraints:

UR1: The work_request_id shall be unique in a population of Work_request.

Informal constraints:

IP1: The work_request_id shall be unique in a population of Work_request in an Organization.

IP2: Each member of Work_request shall have at least one member of Change_request or Start_request in the role of change_or_start_request.

ENTITY work_request
  ABSTRACT SUPERTYPE OF (ONEOF (start_request,change_request))
  SUBTYPE OF (managed_design_object);
    work_request_approval : ee_approval;
    request_recipient     : SET [1:?] OF person_organization;
    work_request_id       : ee_name;
    referenced_product    : SET [1:?] OF ee_product_version;
    request_date          : date_or_date_and_time;
    description           : ee_text;
    reason                : evaluated_characteristic_or_description;
    initiator             : SET [1:?] OF person_organization;
    status                : STRING;
  INVERSE
    implementation_order : SET [0:1] OF work_order FOR referenced_work_request;
  UNIQUE
    ur1 : work_request_id;
END_ENTITY; -- work_request
