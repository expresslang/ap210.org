4.2.765 Reference_functional_unit_assignment_to_part

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Reference_functional_unit_assignment_to_part is a type of Managed_design_object. A Reference_functional_unit_assignment_to_part assigns one specific path in the function reference network graph to an implementation. A Reference_functional_unit_assignment_to_part provides support for exchange of information needed to determine swappability of functions assigned to devices.

The data associated with this application object are the following:

  • functional_design_definition_path
  • implementation
  • of_property
  • path_alias
  • swap_code

4.2.765.1 functional_design_definition_path

The functional_design_definition_path specifies the role of the Design_composition_path for the Reference_functional_unit_assignment_to_part.

4.2.765.2 implementation

The implementation specifies the role of the Physical_unit_usage_view for the Reference_functional_unit_assignment_to_part.

4.2.765.3 of_property

The of_property specifies the role of the Coordinated_characteristc for the Reference_functional_unit_assignment_to_part. There may be more than one of_property for a Reference_functional_unit_assignment_to_part. The of_property need not be specified for a particular Reference_functional_unit_assignment_to_part.

4.2.765.4 path_alias

The path_alias specifies the string that is the identifier for the Reference_functional_unit_assignment_to_part.

4.2.765.5 swap_code

The swap_code specifies the string that is the type of exchange that is valid.

Formal constraints:

UR1: The combination of the implementation and the path_alias shall be unique within a population of Reference_functional_unit_assignment_to_part.

ENTITY reference_functional_unit_assignment_to_part
  SUBTYPE OF (managed_design_object);
    functional_design_definition_path : reference_composition_path;
    implementation                    : physical_unit_or_part_template;
    of_property                       : OPTIONAL SET [1:?] OF
                                        coordinated_characteristic;
    path_alias                        : ee_name;
    swap_code                         : OPTIONAL STRING;
  UNIQUE
    ur1 : implementation, path_alias;
END_ENTITY; -- reference_functional_unit_assignment_to_part
