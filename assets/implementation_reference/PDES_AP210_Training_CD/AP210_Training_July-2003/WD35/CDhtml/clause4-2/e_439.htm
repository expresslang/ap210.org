4.2.446 Interconnect_module_constraint_region

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
An Interconnect_module_constraint_region is a type of Non_feature_shape_element. An Interconnect_module_constraint_region is a region within an Interconnect_module in which a placement constraint on some category of design object is to be met by the design. An Interconnect_module_constraint_region may overlap other Interconnect_module_constraint_regions.

The data associated with this application object are the following:

  • associated_stratum_extent
  • constrained_design_object_category
  • design_specific_purpose
  • keepout
  • non_conformant_interconnect_module_design_object
  • precedent_version_constraint_region
  • requirement
  • scope

4.2.446.1 associated_stratum_extent

The associated_stratum_extent specifies the Inter_stratum_extent as the extent of Stratum for the Interconnect_module_constraint_region. There shall be one or more Inter_stratum_extent for an Interconnect_module_constraint_region. The Inter_stratum_extent shall all be for the same scope.

4.2.446.2 constrained_design_object_category

The constrained_design_object_category specifies either an interconnect_module_cutout, an interconnect_module_fill_area, an interconnect_module_inter_stratum_feature, an interconnect_module_stratum_feature, or an interconnect_module_via as a category for the Interconnect_module_constraint_region. There may be more than one interconnect_module_cutout, interconnect_module_fill_area, interconnect_module_inter_stratum_feature, or interconnect_module_via for an Interconnect_module_constraint_region. The names shall be interpreted as the identification of Application Object categories to be constrained.

4.2.446.3 design_specific_purpose

The design_specific_purpose identifies the purpose the enterprise defined for the Interconnect_module_constraint_region.

4.2.446.4 keepout

The keepout specifies if the meaning of the Interconnect_module_constraint_region is to describe that items shall be in the interior of the element_shape or shall be on the exterior of the element_shape. The value of TRUE states that objects in the constrained_design_object_category are not to be within the element_shape. The value of FALSE states that objects in the constrained_design_object_category are to be only within the element_shape.

4.2.446.5 non_conformant_interconnect_module_design_object

The non_conformant_interconnect_module_design_object specifies either a Component_termination_passage, a Conductive_filled_area, a Conductor, a Cutout, an Inter_stratum_feature, a Laminate_component, a Land, a Non_functional_land, a Stratum_feature, or a Via for the Interconnect_module_constraint_region. There may be more than one Component_termination_passage, Conductive_filled_area, Conductor, Cutout, Inter_stratum_feature, Laminate_component, Land, Non_functional_land, Stratum_feature, or Via for an Interconnect_module_constraint_region.

4.2.446.6 precedent_version_constraint_region

The precedent_version_constraint_region is an inverse relationship that specifies that the Interconnect_module_constraint_region is existence dependent on a specific Interconnect_module_constraint_region as a precedent region. The usage of this relationship is to enable traceability to original Interconnect_module_constraint_region during design iterations.

NOTE: Since the Interconnect_module_constraint_region is dependent on a specific Interconnect_module, it is not possible to use the mechanism of associating one Interconnect_module_constraint_region with several Interconnect_module, each of which is for a different design version. In addition, the non_conformant_interconnect_module_design_object is for objects within the scope, which also precludes direct re-use of the same Interconnect_module_constraint_region. It is expected that only the first Interconnect_module_constraint_region populated would have an associated shape to eliminate the need for carrying redundant shape data.

4.2.446.7 requirement

The requirement specifies the role of the Ee_requirement_occurrence for the Interconnect_module_constraint_region.

4.2.446.8 scope

The scope specifies the Interconnect_module for the Interconnect_module_constraint_region.

Formal constraints:

WR1: The element_shape shall be a Csg_2d_shape.

ENTITY interconnect_module_constraint_region
  SUBTYPE OF (non_feature_shape_element);
    keepout                                          : BOOLEAN;
    non_conformant_interconnect_module_design_object : 
	OPTIONAL SET [1:?] OF interconnect_module_design_object_select;
    SELF\non_feature_shape_element.scope             : interconnect_module;
    design_specific_purpose                          : ee_name;
    requirement                                      : ee_requirement_occurrence;
    associated_stratum_extent                        : SET [1:?] OF inter_stratum_extent;
    constrained_design_object_category               : 
	interconnect_module_design_object_category;
INVERSE
   precedent_version_constraint_region               : SET [0:1] OF 
        non_feature_shape_element_relationship for related_shape_element;
WHERE
  WR1: ('AP210_ARM.CSG_2D_SHAPE' IN 
        TYPEOF(SELF\non_feature_shape_element.element_shape)) OR 
       ('AP210_ARM.BOUND_VOLUME_SHAPE' IN 
         TYPEOF(SELF\non_feature_shape_element.element_shape));
  WR2: NOT(SIZEOF(precedent_version_constraint_region) = 1) OR
        ('AP210_ARM.INTERCONNECT_MODULE_CONSTRAINT_REGION' IN 
        TYPEOF(precedent_version_constraint_region\
              non_feature_shape_element_relationship.relating_shape_element));
END_ENTITY; -- interconnect_module_constraint_region
