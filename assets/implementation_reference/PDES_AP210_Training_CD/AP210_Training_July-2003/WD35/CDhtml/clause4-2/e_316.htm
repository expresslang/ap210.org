4.2.316 Ee_document

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
An Ee_document is a type of Managed_design_object. An Ee_document may be an Ee_specification. An Ee_document is a set of detailed information that describes a product or aspect of a product in detail. The information is in the form of text, tables, and graphics in a human readable language, although the hard copy may be a report from computer interpretable files.

The data associated with this application object are the following:

  • author
  • base_identifier
  • change_date
  • creation_date
  • document_approval
  • document_name
  • key_phrase
  • publisher
  • revision
  • security_code
  • text_usage

4.2.316.1 author

The author specifies the Person who creates an Ee_document. The author need not be specified for a particular Ee_document. There may be more than one Person for an Ee_document.

4.2.316.2 base_identifier

The base_identifier specifies the role of the Ee_document_identifier for the Ee_document.

4.2.316.3 change_date

The change_date specifies the unique Date_and_time when updates are made to an Ee_document. The change_date need not be specified for a particular Ee_document. There may be more than one Date_and_time for an Ee_document.

4.2.316.4 creation_date

The creation_date specifies the Date_and_time when an Ee_document was created.

4.2.316.5 document_approval

The document_approval specifies the Ee_approval of the Ee_document. The document_approval need not be specified for a particular Ee_document.

4.2.316.6 document_name

The document_name specifies the string identifying the Ee_document.

4.2.316.7 key_phrase

The key_phrase specifies the Structured_text that details specific aspects within the text. The key_phrase need not be specified for a particular Ee_document. There may be more than one Structured_text for an Ee_document.

4.2.316.8 publisher

The publisher specifies the Organization or the Person_and_organization responsible for the Ee_document.

4.2.316.9 revision

The revision specifies the string identifying the configuration controlled version of the Ee_document.

4.2.316.10 security_code

The security_code specifies the Security_classification of a particular Ee_document. The security_code need not be specified for a particular Ee_document.

4.2.316.11 text_usage

The text_usage specifies either the Ee_text or the Structured_text for an Ee_document. The text_usage need not be specified for a particular Ee_document. There may be more than one Ee_text or Structured_text for an Ee_document.

ENTITY ee_document
  SUPERTYPE OF (ee_specification)
  SUBTYPE OF (managed_design_object);
    publisher         : person_organization;
    key_phrase        : OPTIONAL SET [1:?] OF structured_text;
    document_approval : OPTIONAL ee_approval;
    base_identifier   : ee_document_identifier;
    change_date       : OPTIONAL SET [1:?] OF date_and_time;
    creation_date     : date_and_time;
    security_code     : OPTIONAL security_classification;
    text_usage        : OPTIONAL plain_or_structured_text;
    author            : OPTIONAL SET [1:?] OF person;
    revision          : ee_name;
    document_name     : ee_name;
UNIQUE
    ur1 : base_identifier, revision;
END_ENTITY; -- ee_document
