4.2.804 Security_classification

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Security_classification is a type of Managed_design_object. A Security_classification is the level of confidentiality that is required for the purpose of product data protection.

The data associated with this application object are the following:

  • classification_date
  • classification_officer
  • classification_level
  • declassification_date
  • security_classification_approval

4.2.804.1 classification_date

The classification_date specifies either the effective Date or the effective Date_and_time for the Security_classification.

4.2.804.2 classification_officer

The classification_officer specifies the Person_and_organization responsible for the Security_classification.

4.2.804.3 classification_level

The classification_level specifies the level for the Security_classification.

4.2.804.4 declassification_date

The declassification_date specifies either the Date or the Date_and_time upon which the Security_classification no longer applies. The declassification_date need not be provided for a particular Security_classification.

4.2.804.5 security_classification_approval

The security_classification_approval specifies the role of the Ee_approval for the Security_classification.

ENTITY security_classification
  SUBTYPE OF (managed_design_object);
    classification_officer           : person_and_organization;
    security_classification_approval : ee_approval;
    classification_date              : date_or_date_and_time;
    classification_level             : ee_name;
    declassification_date            : OPTIONAL date_or_date_and_time;
END_ENTITY; -- security_classification
