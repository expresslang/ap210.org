4.2.756 Promissory_usage_relationship

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Promissory_usage_relationship is a type of Managed_design_object. The Promissory_usage_relationship is the relationship of an item to a higher assembly where the next higher assembly of the item has not been defined.

EXAMPLE: A designer is designing an MCM that will be used on a board that will be part of a system. The designer does not yet know the next higher assembly for the MCM, since the board on which it will be placed has not yet been defined in the product structure for the system. The designer does, however, know the system in which the MCM will be used. Using the Promissory_usage_relationship, the designer can specify that the MCM is promised to be used within the system, without reference to the specific board on which it will reside.

The data associated with this application object are the following:

  • assembly
  • component
  • security_code

4.2.756.1 assembly

The assembly specifies the role of the Physical_unit_design_view for the Promissory_usage_relationship.

4.2.756.2 component

The component specifies the role of the Physical_unit for the Promissory_usage_relationship.

4.2.756.3 security_code

The security_code specifies the role of the Security_classification for the Promissory_usage_relationship. The security_code is optional and need not be supplied for a Promissory_usage_relationship.

Formal constraints:

UR1: The combination of the component and the assembly shall be unique within a population of Promissory_usage_relationship.

ENTITY promissory_usage_relationship
  SUBTYPE OF (managed_design_object);
    component     : physical_unit;
    security_code : OPTIONAL security_classification;
    assembly      : physical_unit_design_view;
  UNIQUE
    ur1 : component, assembly;
END_ENTITY; -- promissory_usage_relationship
