4.2.489 Laminate_text_string_component

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Laminate_text_string_component is a type of Laminate_component and is a type of Array_placement_group. A Laminate_text_string_component conveys a complete one line message to the viewer. A Laminate_text_string_component is composed by associating one or more individual instances of Laminate_text_component to the Laminate_text_string_component using one instance of Sub_assembly_relationship. The Laminate_text_string_component itself does not directly refer to a font definition. The usage_definition inherited from the supertype Assembly_component is unnecessary since this Application Object is defined by the composition of Laminate_text_component contributing to it. The position of the Laminate_text_string_component shall be interpreted as a constraint on the position of the individual components contributing to it. The position of the Laminate_text_string_component shall be interpreted as the insertion point for the message. The Laminate_text_string_component may have an orientation with respect to the design coordinate system, in which case the components composing the Laminate_text_string_component shall have a corresponding orientation. The Axis_placement or Cartesian_transformation_operator_2d which locates the Laminate_text_string_component shall be interpreted as a local coordinate system in order that any orientation shall be applied consistently to all components. An implementation shall ensure that adjacent characters that use different fonts still meet the requirement that Stratum_feature instances are disjoint.

NOTE: If multi-line messages are required, multiple instances of Laminate_text_string_component will be needed, one for each line of the message.

The data associated with this application object are the following:

  • message
  • of_horizontal_justification
  • of_vertical_justification

4.2.489.1 message

The message specifies the message content, a string, for the Laminate_text_string_component.

4.2.489.2 of_horizontal_justification

The horizontal_justification specifies either "left", "right", or "centre" for the beginning point of the message with respect to the position of the Axis_placement or Cartesian_transformation_operator_2d that is the insertion point for the text. The "left" indicates that the left edge of the bounding box of the first character of the message is aligned with the 'Y' axis of the Axis_placement. The "right" indicates that the right edge of the bounding box of the last character of the message is aligned with the 'Y' axis of the Axis_placement. The "centre" indicates that the left edge of the bounding box of the first character of the message and the right edge of the bounding box of the last character of the message are equally displaced on either side of the 'Y' axis of the Axis_placement.

4.2.489.3 of_vertical_justification

The vertical_justification specifies either "top", "bottom", or "centre" for the beginning point of the message with respect to the position of the Axis_placement or Cartesian_transformation_operator_2d that is the insertion point for the text. The "top" indicates that the top edge of the bounding box of the first character of the message is aligned with the 'X' axis of the Axis_placement. The "bottom" indicates that the bottom edge of the bounding box of the last character of the message is aligned with the 'X' axis of the Axis_placement. The "centre" indicates that the top edge of the bounding box of the first character of the message and the bottom edge of the bounding box of the last character of the message are equally displaced on either side of the 'X' axis of the Axis_placement.

ENTITY laminate_text_string_component
  SUBTYPE OF (array_placement_group, laminate_component);
    message                     : message_content;
    of_horizontal_justification : horizontal_justification;
    of_vertical_justification   : vertical_justification;
END_ENTITY; -- laminate_text_string_component
