4.2.605 Part_connected_terminals_definition

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Part_connected_terminals_definition is a type of Managed_design_object. A Part_connected_terminals_definition is the association between two or more Part_terminal that are connected internally to the part. This object serves as a formal requirement statement for connections implemented by a product. It is a topological abstraction in that it does not provide any information stating the variation of properties with respect to product or environmental variability.

NOTE: This object is included to support the case where multiple terminals of a part are connected internally (e.g VCC, Ground) to facilitate mapping between the physical terminal definition and the functional terminal definition.

The data associated with this application object are the following:

  • associated_definition
  • connected_terminals
  • name

4.2.605.1 associated_definition

The associated_definition specifies the Physical_unit_usage_view that is the product definition s for the Part_connected_terminals_definition.

4.2.605.2 connected_terminals

The connected_terminals specifies the role of the Part_terminal for the Part_connected_terminals_definition. There shall be two or more Part_terminal for a Part_connected_terminals_definition.

4.2.605.3 name

The name identifies the Part_connected_terminals_definition.

Formal constraints:

UR1: The combination of the name and associated_definition shall be unique within a population of Part_connected_terminals_definitions.

WR1: All terminals specified by the member of this object shall be on the associated_definition.

WR2: All terminals specified by the member of this object shall be mapped to the same terminal of the functional definition.

ENTITY part_connected_terminals_definition
  SUBTYPE OF (managed_design_object);
    associated_definition : physical_unit_usage_view;
    connected_terminals   : SET [2:?] OF part_terminal;
    name                  : ee_name;

  UNIQUE
    ur1 : name, associated_definition;
WHERE
  WR1 : SIZEOF(QUERY( ct <* connected_terminals |
    NOT(associated_definition = ct\part_feature.associated_definition)
         )) = 0;
END_ENTITY; -- part_connected_terminals_definition
