4.2.896 Test_requirement_occurrence

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Test_requirement_occurrence is a type of Ee_requirement_occurrence that specifies a test related requirement. The test method shall be specified in the requirement_specification inherited from Ee_requirement_occurrence. The Test_requirement_occurrence provides a representation of minimum bounds on behaviour. In order to relate a Test_requirement_occurrence to a product, a Requirement_allocation is required to be exchanged that references this Application Object and the version of the product to which the test requirement is allocated. That Requirement_allocation shall also specify a number of Part_feature_analytical_model_port_assignment in order to specify the measurement access points for the test. It is valid to exchange a Test_requirement_occurrence without a Requirement_allocation Either an Analytical_representation shall be provided or a Functional_specification shall be provided.

NOTES:

  2. This edition of this part of ISO 10303 does not explicitly relate the test requirement to the test setup.
  3. The Test_requirement_occurrence may be used as a reference for Planned_characteristic.
  4. The Test_requirement_occurrence may be a derived requirement from a Design_requirement_occurrence, in which case production capability based margins would cause the actual characteristics to be different between the Design_requirement_occurrence and the Test_requirement_occurrence.
