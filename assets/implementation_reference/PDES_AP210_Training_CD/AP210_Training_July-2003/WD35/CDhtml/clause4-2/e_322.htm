4.2.322 Ee_product_configuration

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
An Ee_product_configuration is a type of Managed_design_object. An Ee_product_configuration is the association of a product version with a model name that enterprise marketing provides to a customer.

The data associated with this application object are the following:

  • configuration_manager
  • item_id
  • model_of_product
  • phase_of_product
  • product_configuration_approval
  • satisfying_product

4.2.322.1 configuration_manager

The configuration_manager specifies either the Organization or the Person_and_organization that manages the Ee_product_configuration.

4.2.322.2 item_id

The item_id specifies the identification of an Ee_product_configuration.

4.2.322.3 model_of_product

The model_of_product specifies the role of the Ee_product_model for the Ee_product_configuration.

4.2.322.4 phase_of_product

The phase_of_product specifies the string that defines the life cycle phase of the Ee_product_configuration.

EXAMPLE: Examples of phase_of_product include "Preliminary design", "Prototype", "Released for Production", "Obsolete for New Manufacture", and"Retired".

4.2.322.5 product_configuration_approval

The product_configuration_approval specifies the Ee_approval that is associated with the Ee_product_configuration.

4.2.322.6 satisfying_product

The satisfying_product specifies the Ee_product_version that has been selected to satisfy a real or perceived customer need for the Ee_product_configuration. The satisfying_product need not be specified for a particular Ee_product_configuration. There may be more than one Ee_product_version for an Ee_product_configuration.

NOTE: In some enterprises the marketing department selects product model nomenclature independent of the existence of actual product versions to satisfy the marketing requirements. This object permits the enterprise to explicitly manage the relationship between model nomenclature and the product version.

Formal constraints:

UR1: The combination of item_id and satisfying_product shall be unique within a population of Ee_product_configurations.

ENTITY ee_product_configuration
  SUBTYPE OF (managed_design_object);
    satisfying_product             : OPTIONAL SET [1:?] OF ee_product_version;
    configuration_manager          : person_organization;
    item_id                        : STRING;
    product_configuration_approval : ee_approval;
    model_of_product               : ee_product_model;
    phase_of_product               : STRING;
  UNIQUE
    ur1 : item_id, satisfying_product;
END_ENTITY; -- ee_product_configuration
