4.2.573 Package

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Package is a type of a Physical_unit_usage_view. A Package may be a Altered_package. A Package describes the physical container of a Packaged_part. A Package includes no more than one package body and one or more package terminals.

EXAMPLES:

  1. Figures pacax through pacto illustrate types of Packages and parameters related to Geometric Dimensioning and Tolerancing supported by this part of ISO 10303. ISO 10303-202 would be used in conjunction with this part of ISO 10303 to define the exchange of the presentation information (i.e., the drawings). This part of ISO 10303 supports the direct application to application exchange of the parametric data illustrated in the drawings.
  2. Figures pof7 through pof1 illustrate parameters and features related to Package orientation for assembly purposes supported by this part of ISO 10303. ISO 10303-202 would be used in conjunction with this part of ISO 10303 to define the exchange of the presentation information (i.e., the drawings).

NOTE: When used for a Packaged_part "white box model", the Part_device is installed in the Package and an electrical connection is defined between each terminal on the installed Part_device and one or more terminals on the Package.

4.2.573.1 body

The body is an inverse relationship that specifies the Package_body that specifies the Package as its associated_definition. There shall be no more than one Package_body for a Package.

4.2.573.2 cutout_required

The cutout_required specifies a derived value of TRUE or FALSE, depending on the existence of the optional attribute maximum_body_height_below_seating_plane. If the attribute exists, the value is TRUE. If the attribute is no present, the value is FALSE.

4.2.573.3 least_lead_length_below_seating_plane

The least_lead_length_below_seating_plane is a Length_data_element that specifies the smallest lead length below the seating plane. The least_lead_length_below_seating_plane need not be specified for a particular Package.

4.2.573.4 least_material_condition_centroid_location

The least_material_condition_centroid_location specifies a Cartesian_point that is the location of the center of mass for the minimum material state for the Package. The least_material_condition_centroid_location need not be specified for a Package.

4.2.573.5 maximum_body_clearance_above_seating_plane

The maximum_body_clearance_above_seating_plane specifies a Length_data_element that is the maximum length between the body and the top of the seating_plane. A maximum_body_clearance_above_seating_plane need not be specified for a Package.

4.2.573.6 maximum_body_clearance_below_seating_plane

The maximum_body_clearance_below_seating_plane specifies a Length_data_element that is the maximum length between the body and the bottom of the seating_plane. A maximum_body_clearance_below_seating_plane need not be specified for a Package.

4.2.573.7 maximum_body_height_above_seating_plane

The maximum_body_height_above_seating_plane is a Length_data_element that specifies the maximum height of the Package above the seating plane. Since there is no attribute describing the maximum terminal height above the seating plane, this attribute shall be interpreted as the furthest point away from the seating plane of either the body or of any Package_terminal above the seating plane.

EXAMPLE: In the case of package to be mounted in a cutout there may only be the surface mounted leads protruding above the seating plane.

4.2.573.8 maximum_body_height_below_seating_plane

The maximum_body_height_below_seating_plane is a Length_data_element that specifies the maximum height of the Package below the seating plane. The maximum_body_height_below_seating_plane need not be specified for a particular Package.

4.2.573.9 maximum_lead_length_below_seating_plane

The maximum_lead_length_below_seating_plane is a Length_data_element that specifies the maximum lead length below the seating plane. The maximum_lead_length_below_seating_plane need not be specified for a particular Package.

4.2.573.10 maximum_material_condition_centroid_location

The maximum_material_condition_centroid_location specifies a Cartesian_point that is the location of the center of mass for the maximum material state for the Package. The maximum_material_condition_centroid_location need not be specified for a Package.

4.2.573.11 minimum_body_clearance_above_seating_plane

The minimum_body_clearance_above_seating_plane specifies a Length_data_element that is the minimum length between the body and the seating_plane. A minimum_body_clearance_below_seating_plane need not be specified for a Package.

4.2.573.12 minimum_body_clearance_below_seating_plane

The minimum_body_clearance_below_seating_plane specifies a Length_data_element that is the minimum length between the body and the seating_plane. A minimum_body_clearance_below_seating_plane need not be specified for a Package.

4.2.573.13 package_accesses

The package_accesses is an inverse relationship that specifies that the existence of the Package is dependent on the existence of the Package_terminal that specifies the Package as its associated_definition. There shall be zero or more Package_terminal for a Package.

4.2.573.14 package_polarity_indication

The package_polarity_indication is an inverse relationship that specifies the Part_feature that specifies the Package as its associated_definition. There shall be no more than one Part_feature for a Package in this role.

4.2.573.15 package_primary_orientation_feature

The package_primary_orientation_feature is an inverse relationship that specifies the Part_feature that specifies the Package as its associated_definition. There shall be no more than one Part_feature for a Package in this role.

4.2.573.16 interface_plane

The interface_plane is an inverse relationship that specifies the Seating_plane that specifies the Package as its scope. The seating_plane simulates the interface surface of an external device.

NOTE: The interface_plane is typically used for connectors, but may be used for modules.

EXAMPLE: A use of interface_plane is in the case of a fiber-optic transceiver that is mounted on a PCB but has a right angle connector to interface with a fiber.

4.2.573.17 package_seating_plane

The package_seating_plane is an inverse relationship that specifies the Seating_plane that specifies the Package as its scope. The seating_plane simulates the mounting surface for the terminals in the case of surface mounted parts and in the case of through-hole technology, an arbitrarily established mounting surface. Terminals are required to approach the seating_plane from the top, which is the side where the values on the Z axis are positive. Package installation is defined to be only from the top of the seating_plane in order to reduce orientation errors between two-dimensional and three-dimensional models in this part of ISO 10303.

4.2.573.18 package_secondary_orientation_feature

The package_secondary_orientation_feature is an inverse relationship that specifies the Part_feature that specifies the Package as its associated_definition. There shall be no more than one Part_feature for a Package in this role.

4.2.573.19 package_tertiary_orientation_feature

The package_tertiary_orientation_feature is an inverse relationship that specifies the Part_feature that specifies the Package as its associated_definition. There shall be no more than one Part_feature for a Package in this role.

4.2.573.20 mounting_technology

The mounting_technology specifies a value of surface_mount or through_hole for the Package. Surface_mount indicates that the Package_terminals have a Connection_zone that is congruent with the package_seating_plane and through_hole indicates the terminals penetrate the package_seating_plane.

Formal constraints:

WR1: Only a Primary_orientation_feature shall be in the role of package_primary_orientation_feature.

WR2: Only a Secondary_orientation_feature shall be in the role of package_secondary_orientation_feature.

WR3: Only a Polarity_indication_feature shall be in the role of package_polarity_indication.

WR4: The package_primary_orientation_feature shall not be the package_secondary_orientation_feature.

WR5: The package_primary_orientation_feature shall not be the package_tertiary_orientation_feature.

WR6: The package_tertiary_orientation_feature shall not be the package_secondary_orientation_feature.

WR7: There shall be a package category associated with the product the package is a definition for.

NOTE:

ENTITY package
  SUPERTYPE OF (altered_package)
  SUBTYPE OF (physical_unit_usage_view);
    mounting_technology                          : mounting_technology_type;
    maximum_seating_plane_installation_offset    : OPTIONAL length_data_element; 
    surface_mount_lead_form                      : OPTIONAL lead_form_type;
    nominal_mounting_lead_pitch                  : OPTIONAL length_data_element; 
    nominal_mounting_lead_span                   : OPTIONAL length_data_element; 
    least_material_condition_centroid_location   : OPTIONAL cartesian_point;
    maximum_material_condition_centroid_location : OPTIONAL cartesian_point;
    maximum_body_height_above_seating_plane      : length_data_element;
    maximum_body_height_below_seating_plane      : OPTIONAL length_data_element;
    maximum_body_clearance_above_seating_plane   : OPTIONAL length_data_element;
    maximum_body_clearance_below_seating_plane   : OPTIONAL length_data_element;
    minimum_body_clearance_above_seating_plane   : OPTIONAL length_data_element;
    minimum_body_clearance_below_seating_plane   : OPTIONAL length_data_element;
    maximum_lead_length_below_seating_plane      : OPTIONAL length_data_element;
    least_lead_length_below_seating_plane        : OPTIONAL length_data_element;
    of_datum_reference_frame                     : OPTIONAL datum_reference_frame;
    of_defining_datum_system_relationship        : OPTIONAL SET [1:3] OF 
                                datum_feature_usage_in_datum_system_relationship;
 of_derived_datum_system_definining_relationship : OPTIONAL SET [1:3] OF
                                        datum_usage_in_datum_system_relationship;


DERIVE
(* 
The total vertical extent of the package calculation:
*)
    maximum_package_total_vertical_extent : length_data_element :=
       max_data_element(
                 add_data_element(maximum_body_height_above_seating_plane,
      				  maximum_body_height_below_seating_plane),
		 add_data_element(maximum_body_height_above_seating_plane,
				  maximum_lead_length_below_seating_plane));
(*
A cutout is required for the package if the package body extends below the
seating plane.  This can be determined by the existence of a value for the
maximum_body_height_below_seating_plane.
*)
    cutout_required : BOOLEAN := 
	 (maximum_body_height_below_seating_plane\data_element.measure_value > 0.0);
    maximum_installed_height : length_data_element :=
       add_data_element(maximum_body_height_above_seating_plane,
                        maximum_seating_plane_installation_offset); 
INVERSE
    body             : SET [0:1] OF package_body FOR associated_definition;
    package_seating_plane                        : 
                  non_feature_shape_element FOR scope;
    interface_plane                              : 
                  SET[0:1] OF non_feature_shape_element FOR scope;
    package_accesses : SET [0:?] OF package_terminal FOR 
                          associated_definition;
    package_polarity_indication : SET [0:1] OF part_feature FOR 
		        associated_definition;
    package_primary_orientation_feature          : SET [0:1] OF part_feature FOR
			associated_definition;
    package_secondary_orientation_feature        : SET [0:1] OF part_feature FOR
			associated_definition;
    package_tertiary_orientation_feature         : SET [0:1] OF part_feature FOR
			associated_definition;
WHERE
 WR1: NOT(EXISTS(package_primary_orientation_feature)) OR
('AP210_ARM.PRIMARY_ORIENTATION_FEATURE' IN TYPEOF(package_primary_orientation_feature));
 WR2: NOT(EXISTS(package_secondary_orientation_feature)) OR
('AP210_ARM.SECONDARY_ORIENTATION_FEATURE' IN TYPEOF(package_secondary_orientation_feature));
 WR3: NOT(EXISTS(package_tertiary_orientation_feature)) OR
('AP210_ARM.TERTIARY_ORIENTATION_FEATURE' IN TYPEOF(package_tertiary_orientation_feature));
(* orientation features shall be referenced by the 
   datum_feature_usage_in_datum_system_relationship
   and shall be datum_features that establish the datum_reference_frame by establishing from
   one to 3 datum_system.
   also, the required datum_usage_in_datum_system_relationship shall be derived from
   the datum_feature_usage_in_datum_system_relationship.
*)
  WR4:'AP210_ARM.SEATING_PLANE' IN TYPEOF(package_seating_plane);
  WR5:'AP210_ARM.SEATING_PLANE' IN TYPEOF(interface_plane);
  WR6:NOT(mounting_technology = surface_mount) OR
         (NOT(EXISTS(maximum_lead_length_below_seating_plane) OR 
              EXISTS(least_lead_length_below_seating_plane)));
  WR7: NOT(EXISTS(maximum_lead_length_below_seating_plane)) OR
        (maximum_lead_length_below_seating_plane\data_element.measure_value > 0.0);
  WR8: NOT(EXISTS(least_lead_length_below_seating_plane)) OR
        (least_lead_length_below_seating_plane\data_element.measure_value > 0.0);
  WR9: (NOT(mounting_technology = surface_mount) AND 
       NOT(EXISTS(surface_mount_lead_form))) OR
       ((mounting_technology = surface_mount) AND
       (EXISTS(surface_mount_lead_form)));
  WR10: package_seating_plane\shape_element.name = 'package seating plane';
  WR11: NOT(EXISTS(interface_plane[1])) OR 
        (interface_plane\shape_element.name = 'interface plane');
  WR12: NOT(EXISTS(interface_plane[1])) OR
        (package_seating_plane <> interface_plane[1]);
END_ENTITY; -- package
