4.2.780 Rule_action

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Rule_action is a type of Managed_design_object. A Rule_action is one of a Rule_adoption, a Rule_change_request, a Rule_creation, a Rule_expiration, a Rule_modification, a Rule_rejection, a Rule_request, a Rule_submission, or a Rule_supersedence. A Rule_action is an assertion that some action has occurred toward management of rules.

The data associated with this application object are the following:

  • subject_action_assignment
  • subject_rule

4.2.780.1 subject_action_assignment

The subject_action_assignment specifies the role of the Participant_date_and_time_assignment for the Rule_action.

4.2.780.2 subject_rule

The subject_rule specifies the Ee_rule_version as the version of an Ee_rule for which some management Rule_action has occurred.

Formal constraints:

UR1: The combination of the subject_action_assignment and the subject_rule shall be unique within a population of Rule_actions.
