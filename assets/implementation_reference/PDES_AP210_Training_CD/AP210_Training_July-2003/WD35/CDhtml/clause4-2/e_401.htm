4.2.401 Functional_unit_definition

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Functional_unit_definition is a type of Ee_product_definition. A Functional_unit_definition is one of a Functional_unit_network_definition, or a Functional_unit_usage_view. A Functional_unit_definition is the characterization of behaviour realized by one or more versions of a part or a product, or a subset of a product. This characterization is applicable to every occurrence of the element modelled.

NOTE: This part of ISO 10303 explicitly supports requirements traceability from a functional definition to a physical definition by providing application objects with the semantics of a forward design flow (i.e., from functional definition to physical definition). In order to use the standard to exchange functional definitions created only to support analysis of existing physical definitions, it is necessary to populate the discipline_id attribute of the supertype Ee_product_definition. The discipline_id of Ee_product_definition specifies the usage of the Functional_unit_definition. Two possible values are "Analysis", "Synthesis". It may not be possible to correlate each Functional_unit to a corresponding Assembly_component when the value is "Analysis". The value of "Synthesis" is suggested when the purpose is not analysis.

The data associated with this application object are the following:

  • analytical_functional_unit_representation

4.2.401.1 analytical_functional_unit_representation

The analytical_functional_unit_representation specifies the Analytical_representation for the computer-interpretable model whose contents are not defined in this part of ISO 10303 for the Functional_unit_definition. The analytical_functional_unit_representation need not be specified for a particular Functional_unit_definition. There may be more than one Analytical_representation for an Functional_unit_definition.
