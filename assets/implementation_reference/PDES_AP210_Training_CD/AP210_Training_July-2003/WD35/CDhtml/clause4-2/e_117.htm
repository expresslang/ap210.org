4.2.117 Component_external_reference

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Component_external_reference is a type of Managed_design_object. A Component_external_reference is an unambiguous reference to a Next_higher_assembly_relationship in a different design.

EXAMPLE: Items in an Interconnect_module design view may need to reference component to assembly relationships in the assembly in order to unambiguously identify source of layout requirements.

The data associated with this application object are the following:

  • design_owner
  • part_number
  • revision_code
  • product_definition_id
  • reference_designation

4.2.117.1 design_owner

The design_owner specifies one role of a string for the Component_external_reference. The design_owner is the organization that maintains the registration document for the part_number for the reference assembly. Usually it is the design authority.

4.2.117.2 part_number

The part_number specifies one role of a string for the Component_external_reference. The part_number identifies the reference assembly in the context of the design_owner.

4.2.117.3 revision_code

The revision_code specifies one role of a string for the Component_external_reference. The revision_code identifies the version of the reference assembly.

NOTE: Usually the revision_code and part number identify the original assembly for which an interconnect is designed.

4.2.117.4 product_definition_id

The product_definition_id specifies one role of a string for the Component_external_reference. The product_definition_id identifies the original Ee_product_definition for the assembly. If not available in the originating system, the string "UNKNOWN" shall be provided.

4.2.117.5 reference_designation

The reference_designation specifies one role of a string for the Component_external_reference. The reference_designation uniquely identifies the externally referenced component in the reference assembly.
