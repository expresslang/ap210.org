4.2.330 Ee_requirement_occurrence

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
An Ee_requirement_occurrence is a type of Managed_design_object. An Ee_requirement_occurrence may be a Requirement_group_occurrence, and additionally may be one of a Constraint_occurrence, a Design_characteristic_occurrence, a Design_requirement_occurrence, a Inspection_requirement_occurrence, a Test_requirement_occurrence, or a Purpose_occurrence and additionally may be one of an Electrical_requirement_occurrence, an Mechanical_requirement_occurrence, an Electromagnetic_requirement_occurrence or a Thermal_requirement_occurrence, and additionally may be one of an Item_restricted_requirement_occurrence or a Layout_spacing_requirement_occurrence. An Ee_requirement_occurrence is a statement of criteria for design or manufacturing that may be derived from more abstract criteria or may be a decomposition of a technical parameter associated wtih a product structure. Complex requirements may be associated with a product specifically for the purpose of managing the requirements.

NOTE: This part of ISO 10303 provides generic categories for enterprise classification of requirements and includes some domain classified requirements.

EXAMPLE: An inspection requirement may be derived from an electromagnetic requirement.

The data associated with this application object are the following:

  • required_analytical_representation
  • required_functional_specification
  • description
  • name
  • hierarchy_context
  • required_characteristic
  • required_coordinated_characteristic
  • required_material
  • required_part
  • requirement_specification

4.2.330.1 required_analytical_representation

The required_analytical_representation specifies the role of the Analytical_representation for the Ee_requirement_occurrence. There may be more than one Analytical_representation for an Ee_requirement_occurrence. The required_analytical_representation need not be specified for a particular Ee_requirement_occurrence.

4.2.330.2 required_functional_specification

The required_functional_specification specifies the role of the Functional_specification for the Ee_requirement_occurrence. There may be more than one Functional_specification for an Ee_requirement_occurrence. The required_functional_specification need not be specified for a particular Ee_requirement_occurrence.

4.2.330.3 description

The description specifies the Ee_text of details of the Ee_requirement_occurrence in a natural language. The description need not be specified for a particular Ee_requirement_occurrence.

4.2.330.4 name

The name specifies the unique string that identifies the Ee_requirement_occurrence.

NOTE: The name should include information identifying the source of the requirement.

4.2.330.5 hierarchy_context

The hierarchy_context is an inverse relationship that specifies the Requirement_composition that specifies the Ee_requirement_occurrence as its element. There shall be no more than one Requirement_composition for an Ee_requirement_occurrence.

4.2.330.6 required_characteristic

The required_characteristic specifies the role of the Characteristic for the Ee_requirement_occurrence. The required_characteristic need not be specified for a particular Ee_requirement_occurrence. There may be more than one Characteristic for an Ee_requirement_occurrence.

4.2.330.7 required_coordinated_characteristic

The required_coordinated_characteristic specifies the role of the Coordinated_characteristic for an Ee_requirement_occurrence. The required_coordinated_characteristic need not be specified for a particular Ee_requirement_occurrence. There may be more than one Coordinated_characteristic for an Ee_requirement_occurrence.

4.2.330.8 required_material

The required_material specifies the role of the Ee_material for the Ee_requirement_occurrence. The required_material need not be specified for a particular Ee_requirement_occurrence.

4.2.330.9 required_part

The required_part specifies the role of the Ee_product for the Ee_requirement_occurrence. The required_part need not be specified for a particular Ee_requirement_occurrence. There may be more than one Ee_product for an Ee_requirement_occurrence.

4.2.330.10 requirement_specification

The requirement_specification specifies the Ee_specification for the source of the Ee_requirement_occurrence. There shall be one or more Ee_specification for an Ee_requirement_occurrence.

Formal constraints:

UR1: The associated_definition shall be unique within the population of Ee_requirement_occurrence.

ENTITY ee_requirement_occurrence
  SUPERTYPE OF (((ONEOF (thermal_requirement_occurrence,
      electromagnetic_requirement_occurrence,
      electrical_requirement_occurrence,
      mechanical_requirement_occurrence) ANDOR ONEOF (
      design_requirement_occurrence,
      inspection_requirement_occurrence,
      test_requirement_occurrence,
      purpose_occurrence,
      constraint_occurrence,
      design_characteristic_occurrence)) ANDOR 
      ONEOF (layout_spacing_requirement_occurrence,
      item_restricted_requirement_occurrence)) ANDOR 
      requirement_group_occurrence)
  SUBTYPE OF (managed_design_object);
    associated_definition               : requirement_definition;
    required_analytical_representation  : OPTIONAL SET [1:?] OF 
                                          analytical_representation;
    required_functional_specification   : OPTIONAL SET [1:?] OF 
                                          functional_specification;
    required_part                       : OPTIONAL SET [1:?] OF ee_product;
    required_coordinated_characteristic : OPTIONAL SET [1:?] OF 
                                          coordinated_characteristic;
    requirement_specification           : SET [1:?] OF ee_specification;
    required_characteristic             : OPTIONAL SET [1:?] OF characteristic;
    required_material                   : OPTIONAL ee_material;
    description                         : OPTIONAL ee_text;
    name                                : ee_name;
  INVERSE
    hierarchy_context : SET [0:1] OF requirement_composition FOR element;
  UNIQUE
   ur1: associated_definition;
END_ENTITY; -- ee_requirement_occurrence
