4.2.891 Terminal_swap_specification

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Terminal_swap_specification is a type of Change_design_object_management_relationship. The Terminal_swap_specification is part of the swapping specification from the previous design definition to the current design definition. It identifies the terminal in the previous design that was connected to a node, and the terminal in the current design that is connected to that node instead.

NOTE: With the pin mapping between the functional and physical representations in the library, this is sufficient information to derive the swapping at the component terminal; however, note that this is in the previous and present design definitions and not in the library functional definition.

The data associated with this application object are the following:

  • current_design_object
  • previous_design_object

4.2.891.1 current_design_object

The current_design_object specifies the Functional_unit_terminal_node_assignment of the terminal node after the swap for the Terminal_swap_specification.

4.2.891.2 previous_design_object

The previous_design_object specifies the Functional_unit_terminal_node_assignment of the terminal node before the swap for the Terminal_swap_specification.

Formal constraints:

WR1: The current_design_object shall not be the previous_design_object.
