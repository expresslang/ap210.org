4.2.609 Part_feature

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Part_feature is a type of Physical_feature_or_part_template. A Part_feature may be one or more of a Part_group_feature, a Part_mounting_feature, a Part_tooling_feature, a Thermal_feature, a Package_terminal, or a Part_terminal; or additionally one of a Primary_orientation_feature, or a Package_body, or an Interconnect_module_surface_feature.

NOTES:

  1. A Part_feature is a distinctive aspect of a Physical_unit that partially supports the choice of that Physical_unit in the design.
  2. The design_view_definition_component_feature and design_view_definition_stratum_concept should only be populated for design archival and should not normally be publicly available as they allow queries against detailed design data.

The data associated with this application object are the following:

  • associated_definition
  • design_view_definition_component_feature
  • design_view_definition_stratum_concept
  • feature_shape
  • name
  • precedent_feature

4.2.609.1 associated_definition

The associated_definition specifies the Physical_unit_usage_view for the targeted Part_feature.

4.2.609.2 design_view_definition_component_feature

The design_view_definition_component_feature is an inverse relationship that specifies the Component_feature_to_physical_usage_view_assignment that specifies the Part_feature as its defined_part_feature. There shall be no more than one Component_feature_to_physical_usage_view_assignment for a Part_feature.

4.2.609.3 design_view_definition_stratum_concept

The design_view_definition_stratum_concept is an inverse relationship that specifies the Stratum_concept_to_physical_usage_view_assignment that specifies the Part_feature as its defined_part_feature. There shall be no more than one Stratum_concept_to_physical_usage_view_assignment for a Part_feature.

4.2.609.4 feature_shape

The feature_shape specifies the role of the Feature_shape_occurrence for the Part_feature. The feature_shape need not be specified for a particular Part_feature. There may be more than one Feature_shape_occurrence for a Part_feature.

4.2.609.5 name

The name specifies the descriptive label recognizable within the electronics community and by a computer system for the Part_feature. The name is a redeclaration of the name attribute inherited from Shape_element and shall be interpreted as the name attribute of shape_aspect.

4.2.609.6 precedent_feature

The precedent_features specifies the role of the Part_feature for the Part_feature. The precedent_feature need not be specified for a particular Part_feature.

Formal constraints:

UR1: The combination of the associated_definition and the name shall be unique within a population of Part_features.

WR1: The shape of features shall be consistent with the shape of the physical_unit_usage_view defined by associated_definition.

WR2: The Part_feature shall not participate in a cyclic relationship with the precedent_feature, either directly or indirectly.

WR3: Each Part_feature shall not be assigned by both a Stratum_concept_to_physical_usage_view_assignment and a Component_feature_to_physical_usage_view_assignment.

ENTITY part_feature
  SUPERTYPE OF (((((((((((part_group_feature ANDOR 
    ONEOF(primary_orientation_feature, 
      secondary_orientation_feature,
      tertiary_orientation_feature,
      package_body)) ANDOR 
      part_tooling_feature) ANDOR 
      thermal_feature) ANDOR 
      part_interface_access_feature) ANDOR 
      part_mating_feature) ANDOR 
      part_mounting_feature) ANDOR 
      part_terminal) ANDOR 
      interconnect_module_surface_feature) ANDOR 
      package_terminal) ANDOR
      package_body_surface) ANDOR 
      polarity_indication_feature)
  SUBTYPE OF (physical_feature_or_part_template);
    associated_definition   : physical_unit_usage_view;
    material_state_change   : OPTIONAL material_state_change_enumeration;
    precedent_feature       : OPTIONAL part_feature;
    SELF\shape_element.name : ee_name;
INVERSE
    design_view_definition_stratum_concept   : 
	SET [0:1] OF stratum_concept_to_physical_usage_view_assignment FOR 
	defined_part_feature;
    design_view_definition_component_feature : 
	SET [0:1] OF component_feature_to_physical_usage_view_assignment FOR 
      defined_part_feature;
    feature_shape         : SET [0:?] OF usage_concept_usage_relationship FOR
    associated_usage;
UNIQUE
    ur1 : SELF\shape_element.name, associated_definition;
END_ENTITY; -- part_feature
