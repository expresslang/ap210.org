4.2.400 Functional_unit

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Functional_unit is a type of Managed_design_object. A Functional_unit is an occurrence of a Functional_unit_definition in the composition of a Functional_unit_network_definition.

EXAMPLE: Figure fu is a schematic of a network definition showing hierarchical relationships, as well as the usage view and network definition for a functional block.

The data associated with this application object are the following:

  • access_mechanisms
  • composed_network
  • definition
  • functional_property
  • reference_designation

4.2.400.1 access_mechanisms

The access_mechanisms is an inverse relationship that specifies that the existence of the Functional_unit is dependent on the existence of the Functional_unit_terminal that specifies the Functional_unit as its accessed_functional_unit. There shall be one or more Functional_unit_terminal for a Functional_unit.

4.2.400.2 composed_network

The composed_network specifies the role of the Functional_unit_network_definition for the Functional_unit.

4.2.400.3 definition

The definition specifies the role of the Functional_unit_definition for the Functional_unit.

4.2.400.4 functional_property

The functional_property specifies the role of the Parameter_assignment for the Functional_unit. There may be more than one Parameter_assignment for the Functional_unit. The functional_property need not be provided for a particular Functional_unit. The functional_property shall be interpreted as overriding the properties supplied as Parameter_assignment for the Functional_unit_definition in the role of definition when the assigned_parameter are identical.

4.2.400.5 reference_designation

The reference_designation specifies the name that identifies the Functional_unit, in the context of the Functional_unit_network_definition.

Formal constraints:

UR1: The combination of the reference_designation and the composed_network shall be unique within a population of Functional_units.

WR1: A network definition shall not define itself.

WR2: Considering the functional_unit_definition entities as nodes in a structural graph, and the functional_unit entity as the links of the graph, the graph thus composed shall be directed and acyclic.

NOTE: A node that is a Functional_unit_definition may be a Functional_unit_network_definition.

ENTITY functional_unit
  SUBTYPE OF (managed_design_object);
    definition            : functional_unit_definition;
    reference_designation : ee_name;
    composed_network      : functional_unit_network_definition;
    functional_property   : OPTIONAL SET [1:?] OF parameter_assignment;
  INVERSE
    access_mechanisms : SET [1:?] OF functional_unit_terminal FOR 
                        accessed_functional_unit;
  UNIQUE
    ur1 : reference_designation, composed_network;
END_ENTITY; -- functional_unit
