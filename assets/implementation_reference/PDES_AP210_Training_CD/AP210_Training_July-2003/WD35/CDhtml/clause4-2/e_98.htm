4.2.98 Characteristic

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Characteristic is a type of Managed_design_object. A Characteristic may be one of a Compound_characteristic, or a Numeric_parameter. A Characteristic is a property of an item.

NOTE: A Characteristic may describe either a physical item, like a realized part, or a description that is unrealizable, such as a simulation model.

The data associated with this application object are the following:

  • property_value
  • reference_document
  • type_name

4.2.98.1 property_value

The property_value specifies either a Data_element, an External_definition, or one of a boolean type, a LOGICAL, or a string type, for the Characteristic.

4.2.98.2 reference_document

The reference_document specifies the Ee_document or a string identifying this Characteristic.

NOTE: This part of ISO 10303 explicitly supports the unambiguous exchange of properties whose names are registered by a registration authority and whose definitions are included in data dictionaries. This part of ISO 10303 includes facilities to unambiguously exchange a wide variety of units including SI, user named, and derived.

4.2.98.3 type_name

The type_name specifies the string for this Characteristic.

Formal constraints:

UR1: The string representing the type_name shall not appear more than once in the reference_document in the role of "type name for Characteristic".

ENTITY characteristic
  SUPERTYPE OF (ONEOF(compound_characteristic,
     numeric_parameter))
  SUBTYPE OF (managed_design_object);
    type_name          : STRING;
    reference_document : ee_document;
    property_value     : numeric_or_non_numeric_select;
END_ENTITY; -- characteristic
