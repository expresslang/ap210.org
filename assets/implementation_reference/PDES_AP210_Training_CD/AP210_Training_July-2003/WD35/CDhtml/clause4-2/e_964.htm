4.2.964 Work_order

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Work_order is a type of Managed_design_object. A Work_order is one of a Change_order or a Start_order, and is a document that authorizes work in the development of an initial or modified design of a product. A Work_order is the result of the processing of one or more Work_request.

The data associated with this application object are the following:

  • additional_data
  • analysis_data
  • authorizing_agreement
  • initiator
  • referenced_work_request
  • release_date
  • resulting_product
  • start_date
  • work_order_approval
  • work_order_id

4.2.964.1 additional_data

The additional_data specifies the Ee_text that provides additional pertinent information that was compiled during the development of the related set of Work_request for the Work_order. The additional_data need not be specified for a particular Work_order.

4.2.964.2 analysis_data

The analysis_data specifies the Ee_text for the result of an assessment performed to validate the design or modification in a related set of Work_requests for the Work_order. The analysis_data need not be specified for a particular Work_order.

4.2.964.3 authorizing_agreement

The authorizing_agreement specifies the role of the Contract for the Work_order. The authorizing_agreement need not be provided for a Work_order.

4.2.964.4 initiator

The initiator specifies either the Person_and_organization or the Organization, that initiates the Work_order. There shall be one or more Person_and_organization or Organization for the Work_order.

4.2.964.5 referenced_work_request

The referenced_work_request specifies the Organization that initiated this Work_order. There shall be one or more Organization for a Work_order.

4.2.964.6 release_date

The release_date specifies either the Date or the Date_and_time upon which the Work_order is released.

4.2.964.7 resulting_product

The resulting_product specifies the Ee_product_version of the objects that are the subject of the Work_order. There shall be one or more Ee_product_version for a Work_order.

4.2.964.8 start_date

The start_date specifies either the Date or the Date_and_time upon which the Work_order is begun.

4.2.964.9 work_order_approval

The work_order_approval specifies the Ee_approval that applies to the Work_order.

4.2.964.10 work_order_id

The work_order_id specifies the identifier of the Work_order.

Formal constraints:

UR1: The work_order_id shall be unique within a population of Work_order.

WR1: The resulting_product and the referenced_work_request shall specify the same Ee_product_version.

ENTITY work_order
  ABSTRACT SUPERTYPE OF (ONEOF (change_order,start_order))
  SUBTYPE OF (managed_design_object);
    referenced_work_request : SET [1:?] OF work_request;
    work_order_id           : ee_name;
    resulting_product       : SET [1:?] OF ee_product_version;
    analysis_data           : OPTIONAL ee_text;
    additional_data         : OPTIONAL ee_text;
    work_order_approval     : ee_approval;
    initiator               : SET [1:?] OF person_organization;
    start_date              : date_or_date_and_time;
    release_date            : date_or_date_and_time;
    authorizing_agreement   : OPTIONAL contract;
  UNIQUE
    ur1 : work_order_id;
END_ENTITY; -- work_order
