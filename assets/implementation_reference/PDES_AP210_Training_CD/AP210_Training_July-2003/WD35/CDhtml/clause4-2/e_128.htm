4.2.128 Component_mounting_clearance_relationship

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Component_mounting_clearance_relationship is a type of Managed_design_object. A Component_mounting_clearance_relationship is the relationship between the features of one component and another in their next assembly, with respect to the clearance between them.

NOTES:

  1. This clearance is required when a specific component requires a minimum distance from the printed circuit board or another specific component.
  2. This clearance is often derived from the mechanism requirements of the factory equipment that assembles the components to their next assembly. The mechanical handling fingers that grasp and place a component require clearance from previously placed components in order to place the second component and release it from the mechanism.

The data associated with this application object are the following:

  • feature_relationship
  • maximum_mounting_clearance
  • minimum_mounting_clearance

4.2.128.1 feature_relationship

The current_component specifies the Component_feature_relationship of the features of the components to be placed for the Component_mounting_clearance_relationship.

4.2.128.2 maximum_mounting_clearance

The maximum_mounting_clearance specifies the Length_data_element, of the maximum distance between the current_component and the previously_placed_component, for the Component_mounting_clearance_relationship. The maximum_mounting_clearance need not be specified for a particular Component_mounting_clearance_relationship. The value shall be interpreted as the specification to be satisfied by the assembly process.

4.2.128.3 minimum_mounting_clearance

The minimum_mounting_clearance specifies the Length_data_element, of the minimum distance between the current_component and the previously_placed_component, for the Component_mounting_clearance_relationship. The value shall be interpreted as the specification to be satisfied by the assembly process.

Formal constraints:

WR1: The feature_relationship shall relate two features in the same design but not of the same component.

WR2: A Datum_plane shall be defined for each of the current_component and of the previously_placed_component.

WR3: The length shall be measured between the two Datum_planes.

WR4: Both components shall be in the same version of the product.

WR5: Each Datum_plane shall be based on physical features referenced by the feature_relationship.
