4.2.256 Default_via_definition

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Default_via_definition is a type of Passage_technology that defines the default shape of the Via.

The data associated with this application object are the following:

  • deposition_material
  • minimum_as_finished_deposition_thickness
  • minimum_as_finished_passage_extent

4.2.256.1 deposition_material

The deposition_material specifies the role of the Ee_material in the Via for the Default_via_definition.

4.2.256.2 minimum_as_finished_deposition_thickness

The minimum_as_finished_deposition_thickness specifies the Length_data_element that is the lower bound for the thickness of the deposited Ee_material for the Default_via_definition.

4.2.256.3 minimum_as_finished_passage_extent

The minimum_as_finished_passage_extent specifies one role of the Length_data_element for the Default_via_definition.

Formal constraints:

WR1: The passage deposition material shall be conductive.

ENTITY default_via_definition
  SUBTYPE OF (passage_technology);
    SELF\passage_technology.deposition_material                      : ee_material;

    SELF\passage_technology.minimum_as_finished_passage_extent       : length_data_element;
END_ENTITY; -- default_via_definition
