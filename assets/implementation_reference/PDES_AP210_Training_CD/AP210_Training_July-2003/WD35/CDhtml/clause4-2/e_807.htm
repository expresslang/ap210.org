4.2.807 Shape_composition_relationship

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Shape_composition_relationship is a type of Geometric_representation_relationship. A Shape_composition_relationship is the association of a Non_feature_shape_definition with its Axis_placement to create a resultant Non_feature_shape_definition.

NOTE: A Shape_composition_relationship is included to support composition of Non_feature and non-physical shapes. Component_2d_location and other types of location Application Objects support composition of assembly or interconnect shapes. Usage_concept_usage_view_relationship supports composition of shapes in the usage view. The name is generic to support extensibility. Mirroring is possible by use of Cartesian_transformation_operator. The use of mirroring should be carefully considered toavoid confusion in receiving systems.

EXAMPLE: A Shape_composition_relationship would be used to transform a keepout shape for a detail into an overall keepout shape. It may be that the detail shape is a Physical_unit_planar_keepout_shape.

The data associated with this application object are the following:

  • related_feature_shape
  • related_feature_shape_placement
  • resultant_feature_shape

4.2.807.1 related_feature_shape

The related_feature_shape specifies one role of the Non_feature_shape_definition for the Feature_shape_composition_relationship. This role name is redeclared from the shape_2 attribute inherited from Geometric_representation_relationship.

4.2.807.2 related_feature_shape_placement

The related_feature_shape_placement specifies the role of the Axis_placement or of the Cartesian_transformation_operator for the Shape_composition_relationship.

4.2.807.3 resultant_feature_shape

The resultant_feature_shape specifies one role of the Non_feature_shape_definition for the Shape_composition_relationship. This role name is redeclared from the shape_1 attribute inherited from Geometric_representation_relationship.
