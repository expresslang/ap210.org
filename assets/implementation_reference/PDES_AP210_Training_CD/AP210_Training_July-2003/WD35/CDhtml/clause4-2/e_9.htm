4.2.10 Altered_packaged_part

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
An Altered_packaged_part is a type of Packaged_part whose used_package is an Altered_package. The material or shape properties are different from the reference Packaged_part.

NOTE: For packaged parts that may have several alternate configurations (e.g., different lead configurations), an "unaltered" configuration can be designated using the base_packaged_part attribute. This base_packaged_part is then referenced by each Altered_packaged_part, along with the used_package, which references the Altered_package for that Altered_packaged_part. This relationship supports the traceability of altered parts, as well as the capability to readily identify potential replacement parts if a specific, altered part is out of stock, for example.

The data associated with this application object are the following:

  • base_packaged_part
  • used_package

4.2.10.1 base_packaged_part

The base_packaged_part specifies the original, unmodified, Packaged_part for the Altered_packaged_part.

4.2.10.2 used_package

The used_package specifies the Altered_package that is the result of preparing the Package by the enterprise for the Altered_packaged_part.

Formal constraints:

WR1: The Package in the role of used_package for the Packaged_part in the role of base_packaged_part shall be the same Package as the Package in the role of package_to_be_prepared for the Altered_package in the role of used_package.

WR2: A Packaged_part shall not be a base_packaged_part for itself, either directly or indirectly.

NOTES:

  1. This is a conceptual description only. It is understood that an enterprise does not disassemble a packaged part to accomplish its preparation. The view taken is that the package is the only item modified.
  2. It is recommended to use Engineering_make_from For functional modifications (e.g., re-programming).
