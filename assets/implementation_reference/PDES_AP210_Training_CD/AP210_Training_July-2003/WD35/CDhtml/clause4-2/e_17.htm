4.2.17 Analytical_model_port

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
An Analytical_model_port is a type of Managed_design_object. An Analytical_model_port may be an Analytical_model_vector_port, or additionally may be a Digital_analytical_model_port. An Analytical_model_port describes an interface point for an Analytical_model.

NOTE: These points are the points at which a simulator allows a user to stimulate an instance of an Analytical_model or to evaluate the response of an instance of an Analytical_model to a stimulus in an experiment. In this context, instance includes the reference to the Analytical_model and the literals required for the parameters. A simulator that enforces conservative laws enforces that the energy entering an instance of an Analytical_model must be the same as that leaving the instance of the Analytical_model. The structure of the port is not visible merely by accessing this information. The subtypes of this object provide more visibility into the underlying structure as appropriate. Members of the subtype tree of this object are populated as needed by an application in order to disclose the appropriate level of detail. The specialization in the subtypes of this object occur as a result of being more specific about certain properties of the port.

The data associated with this application object are the following:

  • accessed_analytical_model
  • distributed_properties
  • nominal_signal_flow_direction
  • port_name
  • port_type
  • unit_of_measure

4.2.17.1 accessed_analytical_model

The accessed_analytical_model specifies the Analytical_model that contains this Analytical_model_port.

NOTES:

  1. Containment in this case explicitly states that if a member of Analytical_model is deleted from the information base then the members of Analytical_model_port that are contained in that member of Analytical_model are also deleted.
  2. Containment in this case also explicitly states that the visibility of the name of an Analytical_model_port is limited to the Analytical_model that is the accessed_analytical_model.

4.2.17.2 distributed_properties

The distributed_properties specifies that the properties are either a distributed or a lumped element approximation for the Analytical_model_port. A value of TRUE means that the properties are distributed for the Analytical_model_port. A value of FALSE means that the properties are not distributed.

NOTE: Multiple modes should be handled by assigning each mode a different port.

4.2.17.3 nominal_signal_flow_direction

The nominal_signal_flow_direction specifies either BIDIRECTIONAL, INPUT_DIRECTION, NOT_APPLICABLE, OUTPUT_DIRECTION, or UNKNOWN_DIRECTION for the Analytical_model_port. The nominal_signal_flow_direction need not be specified for a particular Analytical_model_port.

4.2.17.4 port_name

The port_name specifies the string that represents an identifier, understandable by the end user, for this Analytical_model_port.

NOTE: The port_name is documented in the reference_document for the Analytical_model.

4.2.17.5 port_type

The port_type specifies one of a boolean_property_type, a logical_property_type, a physical_property_type, a numeric_property_type, or a string_property_type for this Analytical_model_port.

4.2.17.6 unit_of_measure

The unit_of_measure specifies the string for the descriptive label for the unit of measure of the value of a property associated with the simulated physical phenomenon exchanged through a member of the Analytical_model_port. The unit_of_measure need not be specified for a particular Analytical_model_port. The representation of the unit_of_measure is in accordance with ISO 10303-41.

Formal constraints:

UR1: The combination of the port_name and the accessed_analytical_model shall be unique within a population of Analytical_model_port.

NOTE:

ENTITY analytical_model_port
  SUPERTYPE OF (analytical_model_vector_port ANDOR digital_analytical_model_port)
  SUBTYPE OF (managed_design_object);
    accessed_analytical_model     : analytical_model;
    port_name                     : ee_name;
    unit_of_measure               : OPTIONAL STRING;
    port_type                     : model_parameter_type;
    distributed_properties        : BOOLEAN;
    nominal_signal_flow_direction : OPTIONAL signal_flow_direction;
  UNIQUE
    ur1 : port_name, accessed_analytical_model;
END_ENTITY; -- analytical_model_port
