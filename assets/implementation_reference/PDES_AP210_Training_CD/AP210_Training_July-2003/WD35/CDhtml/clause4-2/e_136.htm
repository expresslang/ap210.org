4.2.136 Component_swap_specification

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Component_swap_specification is a type of Change_design_object_management_relationship. The Component_swap_specification specifies which component has been swapped for which other component in a design; both the swapped-out and the swapped-in components are specified.

NOTE: The Component_swap_specification supports the component identification portion of the gate swap specification. It is necessary when components, such as the microcircuits known as U1, U2, etc. are being swapped.

The data associated with this application object are the following:

  • current_design_object
  • previous_design_object

4.2.136.1 current_design_object

The current_design_object specifies the Design_functional_unit_allocation_to_assembly_component for the resulting design object after swapping, for the Component_swap_specification.

4.2.136.2 previous_design_object

The previous_design_object specifies the Design_functional_unit_allocation_to_assembly_component for the original design object before swapping, for the Component_swap_specification.
