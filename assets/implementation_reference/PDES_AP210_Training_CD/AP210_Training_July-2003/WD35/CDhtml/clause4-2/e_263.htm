4.2.263 Design_functional_unit_allocation_to_assembly_component

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Design_functional_unit_allocation_to_assembly_component is a type of Managed_design_object. A Design_functional_unit_allocation_to_assembly_component may be a Transmission_line_functional_unit_to_routed_printed_component_allocation. A Design_functional_unit_allocation_to_assembly_component is the relationship between the Design_composition_path and the Assembly_component that performs the required function.

The data associated with this application object are the following:

  • functional_design_definition_path
  • global_swappable
  • implementation
  • local_swappable
  • of property
  • path_alias
  • swap_code

NOTE: Frozen_assignment is used to indicate that the Design_functional_unit_allocation_to_assembly_component relationship is approved and can not be changed during further design activity without additional formal action.

4.2.263.1 functional_design_definition_path

The functional_design_definition_path specifies the role of the Design_composition_path for the Design_functional_unit_allocation_to_assembly_component.

4.2.263.2 global_swappable

The global_swappable specifies a boolean value that indicates if occurrences of the associated path can be exchanged within the context of an Ee_product being designed.

4.2.263.3 implementation

The implementation specifies the role of the Assembly_component for the Design_functional_unit_allocation_to_assembly_component.

NOTE: A Design_functional_unit_allocation_to_assembly_component approval may be indicated by including a complex instance consisting of Design_functional_unit_allocation_to_assembly_component and Frozen_assignment.

4.2.263.4 local_swappable

The local_swappable specifies a boolean value that indicates if occurrences of the associated path can be exchanged within the context of an Assembly_component.

4.2.263.5 of_property

The of_property specifies the role of the Coordinated_characteristc for the Design_functional_unit_allocation_to_assembly_component. There may be more than one of_property for a Design_functional_unit_allocation_to_assembly_component. The of_property need not be specified for a particular Design_functional_unit_allocation_to_assembly_component.

4.2.263.6 path_alias

The path_alias specifies the string that is the identifier in the context of the implementation for the Design_functional_unit_allocation_to_assembly_component.

EXAMPLE: "A", "B", "C", "D", are typical path_alias for a quad NAND gate part.

4.2.263.7 swap_code

The swap_code specifies the string that is the type of replacement that is valid.

Formal constraints:

UR1: The combination of implementation and path_alias shall be unique within a population of Design_functional_unit_allocation_to_assembly_component.

    of_property                       : OPTIONAL SET [1:?] OF coordinated_characteristic;

    swap_code                         : STRING;
  UNIQUE
    ur1 : implementation, path_alias;
