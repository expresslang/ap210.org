4.2.860 Stratum_feature_styled_area_representation

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Stratum_feature_styled_area_representation is a type of Managed_design_object. A Stratum_feature_styled_area_representation is the association between one or more of a Stratum_feature and one of a Fill_area_occurrence, or a Styled_curve_occurrence. The Stratum_feature_styled_area_representation is intended for use where legacy data may need to be exchanged via this Standard or where complex patterns of Stratum_feature are to be created. The Stratum_feature_styled_area_representation specifies parametric data to be input to an algorithm. The algorithm output shall be the individual Stratum_feature shapes. Once the individual Stratum_feature shapes are available, they may be used if explicit shape data is required by a receiving processor. If the individual shapes are provided, the attribute derived_planar_shape shall be populated to identify the derived shapes. The units used in the Stratum_feature_styled_area_representation shall be those specified in the global geometric context for the Interconnect_module design as specified by Cartesian_coordinate_system.

NOTE: A Stratum_feature_styled_area_representation is not intended to be used to represent a complex Stratum_feature resulting from multiple design intent modifications to a basic shape. The other types of Stratum_feature shape representation should be used to convey that information.

EXAMPLE: The Stratum_feature_styled_area_representation for a "dot" pattern used in manufacturing will result in multiple Stratum_feature for one instance of Stratum_feature_styled_area_representation. Legacy vector formats that "paint" filled areas using a successive approximation application of a constant width curve may be supported with Stratum_feature_styled_area_representation using the Styled_curve_occurrence option. If the curve width varies, use of multiple Design_intent_modification_styled_area_representation may be appropriate.

The data associated with this application object are the following:

  • derived_planar_shape
  • represented_stratum_feature
  • shape_styled_area_representation

4.2.860.1 derived_planar_shape

The derived_planar_shape specifies the role of the Stratum_feature_planar_shape for the Stratum_feature_styled_area_representation. There may be more than one Stratum_feature_planar_shape for the Stratum_feature_styled_area_representation if the Stratum_feature_styled_area_representation is discontiguous. The Stratum_feature_planar_shape need not be supplied for the Stratum_feature_styled_area_representation.

4.2.860.2 represented_stratum_feature

The represented_stratum_feature specifies the role of the Stratum_feature for the Stratum_feature_styled_area_representation. There may be more than one Stratum_feature for the Stratum_feature_styled_area_representation if the Stratum_feature_styled_area_representation is discontiguous.

NOTE: More than one Stratum_feature is required if the shape of the Stratum_feature_styled_area_representation is not a contiguous shape, since Stratum_feature is required to be a single contiguous shape.

4.2.860.3 shape_styled_area_representation

The shape_styled_area_representation specifies one of a Fill_area_occurrence, or a Styled_curve_occurrence as the representation for the Stratum_feature_styled_area_representation.
