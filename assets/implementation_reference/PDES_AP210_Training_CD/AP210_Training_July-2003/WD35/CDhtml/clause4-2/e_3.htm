4.2.3 Add_design_object_management_relationship

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
An Add_design_object_management_relationship is a type of Design_object_management_relationship.

NOTE: Together with Change_design_object_management_relationship and Delete_design_object_management_relationship, Add_design_object_management_relationship supports the exchange of changes to a design by the use of a data set containing only (1) the information necessary to identify the design (e.g., an Ee_product_definition or an Ee_product_version), which is assumed to be already known to the receiving system, and (2) the design objects that have been added, changed, or deleted. Note that all the quality and structural requirements defined in this part of ISO 10303 may also apply to the resultant new design iteration.

The data associated with this application object are the following:

  • current_design_object

4.2.3.1 current_design_object

The current_design_object specifies the role of the Managed_design_object for the Add_design_object_management_relationship.

Formal constraints:

UR1: The combination of the current_design inherited from the supertype Design_object_management_relationship and the current_design_object shall be unique within a population of Add_design_object_management_relationship.

NOTE: 
*)
ENTITY add_design_object_management_relationship
  SUBTYPE OF (design_object_management_relationship);
    current_design_object : managed_design_object;
  UNIQUE
    ur1 : current_design_object, current_design;
END_ENTITY; -- add_design_object_management_relationship
