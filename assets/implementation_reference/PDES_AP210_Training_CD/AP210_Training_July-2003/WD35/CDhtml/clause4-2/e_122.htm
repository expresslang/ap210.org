4.2.122 Component_feature_to_layout_feature_relationship

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Component_feature_to_layout_feature_relationship is a type of Managed_design_object. A Component_feature_to_layout_feature_relationship may be a Component_based_annotation_symbol_placement_relationship or a Component_based_annotation_text_relationship. A Component_feature_to_layout_feature_relationship is the association between a component or feature of a component and a layout feature, based upon a requirement. All features related with this relationship shall be in the same design unless the reference is to a Component_external_reference or to a Component_feature_external_reference.

The data associated with this application object are the following:

  • related_stratum_concept
  • relating_feature
  • requirement

4.2.122.1 related_stratum_concept

The related_stratum_concept specifies either an Inter_stratum_feature, a Laminate_component, a Stratum, a Stratum_feature, or a Stratum_surface to which the relating_feature is related for the Component_feature_to_layout_feature_relationship.

4.2.122.2 relating_feature

The relating_feature specifies an Assembly_component or a Component_feature or a Component_feature_external_reference or a Component_external_reference for the Component_feature_to_layout_feature_relationship.

4.2.122.3 requirement

The requirement specifies the Ee_requirement_occurrence that causes this Component_feature_to_layout_feature_relationship to be needed. The requirement need not be specified for a particular Component_feature_to_layout_feature_relationship.
