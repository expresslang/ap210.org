4.2.549 Model_parameter

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Model_parameter is a type of Managed_design_object. A Model_parameter is a formally declared variable of the Analytical_model provided for an external application to populate at execution time in a computing environment. The base type of the variable is defined by the base type of the default_value or valid_range data.

EXAMPLE: In Spice, temperature is a Model_parameter that may be set at the execution time.

The data associated with this application object are the following:

  • default_value
  • reference_document
  • type_name
  • unit_of_measure
  • valid_range

4.2.549.1 default_value

The default_value specifies either a Characteristic or a Coordinated_characteristic for the parameter. The default_value need not be specified for a particular Model_parameter. When a collection of data types needs to be defined, so that an instance of the collection may be referenced, the Coordinated_characteristic is used. The External_definition specifies the reference_document to define the semantics of the collection.

4.2.549.2 reference_document

The reference_document specifies either an Ee_document or an identifier for the standard document that contains the reference for the Model_parameter. The reference_document shall define the mapping between the parameter_type string values and the semantics to be applied by a simulator, except for the case of physical_property_type which is predefined in this part of ISO 10303.

4.2.549.3 type_name

The type_name specifies the string used as the human-interpretable type name for the Model_parameter.

4.2.549.4 unit_of_measure

The unit_of_measure specifies the string used as the descriptive label for the unit of measure associated with the Model_parameter. The unit_of_measure need not be specified for a particular Model_parameter. The representation of units described in ISO 10303-41 shall be used.

4.2.549.5 valid_range

The valid_range specifies the Coordinated_characteristic of the Model_parameter. The valid_range need not be specified for a particular Model_parameter. There may be more than one Coordinated_characteristic for a Model_parameter. The valid_ranges need not be contiguous.

NOTE: The base units of the valid_range and default_value are the same type.

Formal constraints:

UR1: The combination of the reference_document and the type_name shall be unique within a population of Model_parameter.

ENTITY model_parameter
  SUPERTYPE OF (datum_difference_based_model_parameter)
  SUBTYPE OF (managed_design_object);
    type_name          : STRING;
    unit_of_measure    : OPTIONAL STRING;
    reference_document : ee_document;
    valid_range        : OPTIONAL SET [1:?] OF coordinated_characteristic;
    default_value      : OPTIONAL scalar_or_coordinated_characteristics;
UNIQUE
  ur1: type_name, reference_document;
END_ENTITY; -- model_parameter
