4.2.314 Ee_approval

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
An Ee_approval is a type of Managed_design_object. An Ee_approval is the dated indication that an Organization or a Person_and_organization has reviewed and approved an occurrence of something.

The data associated with this application object are the following:

  • approval_status
  • authorizer
  • purpose
  • signoff_date

4.2.314.1 approval_status

The approval_status specifies either approved, disapproved, not_yet_approved, or withdrawn as the state of consent applied to an aspect of product data as indicated by the Ee_approval.

4.2.314.2 authorizer

The authorizer specifies either a Person_and_organization or an Organization that authorizes the Ee_approval. There shall be one or more Person_and_organization or Organization for an Ee_approval.

4.2.314.3 purpose

The purpose specifies the text that provides the reason for the Ee_approval.

EXAMPLE: Typical values include "preliminary", "detailed", or"final".

4.2.314.4 signoff_date

The signoff_date specifies either the Date or the Date_and_time of the Ee_approval.

Formal constraints:

WR1: There shall be something to approve for approval to exist.
