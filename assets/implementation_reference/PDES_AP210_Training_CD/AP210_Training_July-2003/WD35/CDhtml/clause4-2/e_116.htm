4.2.116 Component_based_annotation_text_relationship

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Component_based_annotation_text_relationship is a type of Component_feature_to_layout_feature_relationship. A Component_based_annotation_text_relationship is the information that specifies an explicit text message based on the component, where the message content is either the value of the reference to the component's locator or the value of the terminal pin identifier.

NOTE: This information permits product data stored in standard exchange files to serve as messages that are presented to the user in the medium of the Stratum, such as the text of the locator reference or the terminal pin name. This information is not intended to be populated for general etched characters on the board.

The data associated with this application object are the following:

  • message_type
  • related_stratum_concept

4.2.116.1 message_type

The message_type specifies either a component locator reference or a terminal locator for the Component_based_annotation_text_relationship. The message_type is a discriminator for determining which data to use from the attribute relating_feature that is inherited from the Application Object Component_feature_to_layout_feature_relationship . The useful options are either Component_external_reference or Component_feature_external_reference. Either of these options will require a query based on the attribute assembly_design_requirement of the design which this Component_feature_to_layout_feature_relationship is helping to define. For the component locator reference option, two additional queries are required, one to determine the Next_higher_assembly_relationship and the second to determine the Assembly_component. The component locator reference is the attribute reference_designation of the returned instance of Next_higher_assembly_relationship. The string content of that attribute shall be the string content of the message of the related_stratum_concept. For the terminal locator option, additional queries are necessary, continuing with the Assembly_component and terminating at the relevant Part_terminal_external_reference with an intermediate query of Component_feature. The string content of the name attribute of Part_terminal_external_reference shall be the string content of the message of the related_stratum_concept.

NOTES:

  1. The attribute message_type is not a value of data. It is a declaration of the type of Component_based_annotation_text_relationship.
  2. The data contained in the reference_designation attribute in a member of Next_higher_assembly_relationship is an instance of a message of one type.
  3. The data contained in the name attribute in a member of Package_terminal is an instance of a message of the second type.

4.2.116.2 related_stratum_concept

The related_stratum_concept specifies the role of the Laminate_text_string_component for the Component_based_annotation_text_relationship. There is no explicit geometric relationship defined in this part of ISO 10303 between the design element referenced by the message presented by the related_stratum_concept and the related_stratum_concept itself. Only the related_stratum_concept is positioned in the current design. The component or feature referenced by the message in the attribute related_stratum_concept is defined in a different design that is an Assembly_module.

NOTE: This is the actual physical message, as realized by the fabrication process. The position of the corresponding elements in the two designs are usually defined by an end user using a layout tool which presents a view of the data from the assembly in the layout tool for design efficiency.
