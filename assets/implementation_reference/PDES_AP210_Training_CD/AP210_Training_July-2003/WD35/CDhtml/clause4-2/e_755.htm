4.2.755 Promissory_usage_in_product_model_relationship

A Promissory_usage_in_product_model_relationship is a type of Managed_design_object. The Promissory_usage_in_product_model_relationship is the relationship of an item to a higher assembly where the part number of the higher level assembly has not been defined.

EXAMPLE: A designer is designing an MCM that will be used on a board that will be part of a system. The designer does not yet know the part number for the system or that information is held in another business system. The designer does, however, know the equipment type of the system in which the MCM will be used. Using the Promissory_usage_in_product_model_relationship, the designer can specify that the MCM is promised to be used within the system, without reference to the part number of the system.

The data associated with this application object are the following:

  • product_model
  • component
  • security_code

4.2.755.1 assembly

The product_model specifies the role of the Ee_product_model for the Promissory_usage_in_product_model_relationship.

4.2.755.2 component

The component specifies the role of the Ee_product_definition for the Promissory_usage_in_product_model_relationship.

4.2.755.3 security_code

The security_code specifies the role of the Security_classification for the Promissory_usage_in_product_model_relationship. The security_code is optional and need not be supplied for a Promissory_usage_in_product_model_relationship.

ENTITY promissory_usage_in_product_model_relationship
  SUBTYPE OF (managed_design_object);
    component     : ee_product_definition;
    security_code : OPTIONAL security_classification;
    product_model : ee_product_model;
END_ENTITY; -- promissory_usage_in_product_model_relationship
