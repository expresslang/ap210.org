4.2.825 Signal_specification

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Signal_specification is a type of Managed_design_object. A Signal_specification is a bounded behavioural characteristic as measured between two terminals of a Functional_unit in a controlled environment. The characterization of the behaviour of the functional_unit_definition is in terms of the representations of the Signals that exist at the interface of the instance of the functional_unit_definition with the network within which the instance is embedded.

The data associated with this application object are the following:

  • characterization_network
  • characterizing_functional_unit
  • maximum_characteristic
  • measurement_access_mechanism
  • measurement_condition
  • minimum_characteristic
  • reference_access_mechanism

4.2.825.1 characterization_network

The characterization_network specifies the Functional_unit_network_definition in which an instance of a usage view of the Functional_unit_definition being characterized is embedded for this Signal_specification. In this part of ISO 10303, the characterization_network shall include all required simulated stimulus sources and measurement sensors.

EXAMPLE: Examples of characterization_network types include test bench (VHDL), test fixture, test circuit.

4.2.825.2 characterizing_functional_unit

The characterizing_functional_unit specifies the Functional_unit being characterized by the Signal_specification.

4.2.825.3 maximum_characteristic

The maximum_characteristic specifies the upper bound for the valid range of the Signal_property for the Signal_specification.

4.2.825.4 measurement_access_mechanism

The measurement_access_mechanism specifies the role of the Functional_unit_terminal for the Signal_specification.

4.2.825.5 measurement_condition

The measurement_condition specifies the Test_specification under which the measurements are valid, including allowed ranges for stimulus conditions, for the Signal_specification.

NOTE: The measurement_condition may specify data using text or graphics.

4.2.825.6 minimum_characteristic

The minimum_characteristic specifies the lower bound for the valid range of the Signal_property for the Signal_specification.

4.2.825.7 reference_access_mechanism

The reference_access_mechanism specifies the Functional_unit_terminal on the Functional_unit being characterized by this Signal_specification. The reference_access_mechanism need not be specified for a particular Signal_specification. The purpose of the reference_access_mechanism is to provide a reference facility for measurements. The reference_access_mechanism need not be provided.

EXAMPLE: The reference_access_mechanism need not be supplied for evaluation of non-causal systems such as s- or z-plane characterized control system elements or digital elements. There is no assumption in this part of ISO 10303 that reference_access_mechanism is related to any standard reference node, for example, "0" in certain continuous system simulators. Example names of the terminal specified by reference_access_mechanism include Ground, Analogue ground, Digital ground, Trigger in, Gate out.

Formal constraints:

WR1: The External_definition in the role of signal_property_category specified by signal_property in the role of maximum_characteristic shall be the same as the External_definition in the role of signal_property_category specified by signal_property in the role of minimum_characteristic.

WR2: The measurement_access_mechanism shall access an occurrence of the definition being characterized.

WR3: The measurement_access_mechanism shall not be the reference_access_mechanism.

WR4: The network in which the functional_unit_definition being characterized is instantiated shall not be the network being characterized.

WR5: The characterizing_functional_unit shall help compose the characterization_network.

WR6: The measurement_access_mechanism shall be an access_mechanism for the characterizing_functional_unit.

WR7: The reference_access_mechanism, if provided, shall be an access_mechanism for the characterizing_functional_unit.

ENTITY signal_specification
  SUBTYPE OF (managed_design_object);
    measurement_condition          : test_specification;
    measurement_access_mechanism   : functional_unit_terminal;
    reference_access_mechanism     : OPTIONAL functional_unit_terminal;
    minimum_characteristic         : signal_property;
    maximum_characteristic         : signal_property;
    characterization_network       : functional_unit_network_definition;
    characterizing_functional_unit : functional_unit;
END_ENTITY; -- signal_specification
