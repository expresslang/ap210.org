4.2.779 Routed_printed_component

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Routed_printed_component is a type of Printed_component.

EXAMPLE: An example of a Routed_printed_component is a meandering stripline.

The data associated with this application object are the following:

  • component_extent
  • routed_shape
  • usage_definition

4.2.779.1 component_extent

The component_extent specifies the role of the Inter_stratum_extent for the Routed_printed_component.

4.2.779.2 routed_shape

The routed_shape specifies the role of the Feature_shape_definition for the Routed_printed_component. The routed_shape may be represented as the result of an extrusion process, where the usage_definition is extruded along the path of the route.

NOTE: This representation is not intended to reflect physical processing.

NOTE: Usually there is one stratum_feature_implementation for each Design_layer_stratum used in the implementation.

4.2.779.3 usage_definition

The usage_definition specifies the role of the Printed_part_cross_section_template for the Routed_printed_component. The usage_definition provides a cross-sectional definition that shall be interpreted as a constant cross-section along the path of the route.
