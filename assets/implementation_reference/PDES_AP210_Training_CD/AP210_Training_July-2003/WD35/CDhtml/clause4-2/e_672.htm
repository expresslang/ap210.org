4.2.672 Physical_unit_usage_view

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Physical_unit_usage_view is a type of Physical_unit. Each Physical_unit_usage_view may be one of an Assembly_module_usage_view, a Bare_die, an Interconnect_module_usage_view, a Package, or a Packaged_part. A Physical_unit_usage_view identifies a Physical_unit as a part in its next assembly with sufficient detail to allow its use but without sufficient detail to permit reproduction of the Physical_unit. The product version referenced by the version attribute inherited from Ee_product_definition need not reference the same product version referenced by the design view.

NOTES:

  1. The product information necessary for a Physical_unit_usage_view is much less than that necessary to design, manufacture, or verify its performance from first principles. Terminals are defined only in the usage view of an item in this part of ISO 10303.
  2. The usage view and design view of product contain similar concepts. It is the responsibility of the design organization to create the mapping between the usage view and the design view. As the design evolves to meet the requirements, the design view will reference different versions than the usage view.
  3. In a drawing based exchange, an interface control drawing would contain the same information found in a usage view. Industry standards describing the information represented by interface control drawings could migrate to computer interpetable standards exchange using this standard.

EXAMPLE: Examples of information provided in a usage view include outline shape, functional block diagram, terminal identification, package orientation features, mounting features, interface models, pin mapping.

The data associated with this application object are the following:

  • approved_part

4.2.672.1 approved_part

The approved_part specifies the Supplied_product_version that satisfies the intended application of a certain Physical_unit_usage_view. The approved_part need not be specified for a particular Physical_unit_usage_view. There may be more than one Supplied_product_version for a Physical_unit_usage_view.
