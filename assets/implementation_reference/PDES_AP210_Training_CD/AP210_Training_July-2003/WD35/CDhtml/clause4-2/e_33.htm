4.2.33 Assembly_bond_definition

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
An Assembly_bond_definition is a type of Managed_design_object. An Assembly_bond_definition is the definition of the joint that uses bulk material to bond together portions of the assembly.

NOTE: This part of ISO 10303 provides the ability to define the typical shape of the bond through the use of Default_assembly_bond_shape.

The data associated with this application object are the following:

  • analytical_assembly_bond_representation
  • associated_definition
  • bond_category
  • bond_material
  • bonded_feature_1
  • bonded_feature_2
  • default_shape
  • name

4.2.33.1 analytical_assembly_bond_representation

The analytical_assembly_bond_representation specifies the role of Analytical_representation for the Assembly_bond_definition. There may be one or more analytical_assembly_bond_representation for Asssembly_bond_defintion. The analytical_assembly_bond_representation need not be specified for a particular Assembly_bond_definition.

4.2.33.2 associated_definition

The associated_definition specifies the role of the Externally_defined_product_definition for the Assembly_bond_definition. The associated_definition is only provided to support identification of the product classification.

4.2.33.3 bond_category

The bond_category specifies the role of the External_definition for the Assembly_bond_definition.

NOTE: This part of ISO 10303 does not predefine the classification of bonds for assembly.

EXAMPLE: Some typical types are wire bond, solder, and conductive adhesive.

4.2.33.4 bond_material

The bond_material specifies the role of the Ee_material for the Assembly_bond_definition. The bond material may be a composition.

4.2.33.5 bonded_feature_1

The bonded_feature_1 specifies one role of Part_feature for Assembly_bond_definition. The bonded_feature_1 defines one of the Part_feature that will be utilized by Assembled_with_bonding.

4.2.33.6 bonded_feature_2

The bonded_feature_2 specifies one role of Part_feature for Assembly_bond_definition. The bonded_feature_2 defines the second Part_feature that will be utilized by Assembled_with_bonding.

4.2.33.7 default_shape

The default_shape is an inverse relationship that specifies that there shall be no more than one Default_assembly_bond_shape that specifies the Assembly_bond_definition as its characterized_bond_definition. The default_shape need not be specified for a particular Assembly_bond_definition.

4.2.33.8 name

The name specifies the string identifying the Assembly_bond_definition.

Formal constraints:

UR1: The name shall be unique within a population of Assembly_bond_definitions.

UR2: The item that is bonded_feature_1 shall not be bonded_feature_2.

WR1: There shall be a product category of 'technology specific model' assigned to the product this assembly_bond_definition is associated with.

WR2: The associated_definition shall not be another subtype of Ee_product_definition.

ENTITY assembly_bond_definition
  SUBTYPE OF (managed_design_object);
    analytical_assembly_bond_representation : 
	OPTIONAL SET [1:?] OF analytical_representation;
    associated_definition                   : externally_defined_ee_product_definition;
    bond_category                           : external_definition;
    bond_material                           : ee_material;
    bonded_feature_1                        : part_feature;
    bonded_feature_2                        : part_feature;
    name                                    : ee_name;
  INVERSE
    default_shape : SET [0:1] OF default_assembly_bond_shape FOR 
                    characterized_bond_definition;
  UNIQUE
    ur1 : name;
WHERE
 WR1: SIZEOF (QUERY( prpc <* 
  USEDIN(associated_definition\ee_product_definition.version.versioned_item,
       'AP210_ARM.EE_PRODUCT_RELATED_PRODUCT_CATEGORY.OF_PRODUCT') | 
              (prpc.name = 'technology specific model')
             )) >= 0;
END_ENTITY; -- assembly_bond_definition
