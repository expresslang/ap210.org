4.2.191 Connectivity_allocation

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Connectivity_allocation is a type of Managed_design_object. A Connectivity_allocation is the relationship between a functional connectivity requirement and its implementation. The collection of functional terminals in the functional decomposition that are connected is allocated to one equivalent physical node. That equivalent physical node is an implementation in the Connectivity_allocation.

NOTE: This part of ISO 10303 supports the capability to specify the paths through the functional decomposition that are reachable via terminals associated with given nodes. This is a statement of transitive closure for connectivity across the decomposition. This part of ISO 10303 does not separately represent a fully resolved functional node.

The data associated with this application object are the following:

  • connectivity_requirement
  • implementation

4.2.191.1 connectivity_requirement

The connectivity_requirement specifies the role of the Aggregate_connectivity_requirement for the Connectivity_allocation.

NOTE: The connectivity_requirement effectively specifies one equivalent functional node.

4.2.191.2 implementation

The implementation specifies the role of the Physical_connectivity_definition for the Connectivity_allocation.

Formal constraints:

UR1: The implementation shall be unique within a population of Connectivity_allocations.

NOTE:

ENTITY connectivity_allocation
  SUBTYPE OF (managed_design_object);
    implementation           : physical_connectivity_definition;
    connectivity_requirement : aggregate_connectivity_requirement;
  UNIQUE
    ur1 : implementation;
END_ENTITY; -- connectivity_allocation
