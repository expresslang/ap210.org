4.2.439 Interconnect_component_join_terminal

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
An Interconnect_component_join_terminal is a type of Component_terminal that is some aspect of a generic Assembly_component participating in connectivity for a Pca. This Interconnect_component_join_terminal is defined in-situ.

EXAMPLES:

  1. An example of an Interconnect_component_join_terminal is a mechanical conductive fastener that attaches a flexible wire lead lug to the pcb. Current flows through the fastener because it is part of the conduction path for a signal.
  2. A plain wire is considered to be a mechanical component as well. There is no Part_feature associated with it. This is the opposite end of the wire from that defined for the same wire in Interconnect_component_interface_terminal.

The data associated with this application object are the following:

  • connection
  • name

4.2.439.1 connection

The connection specifies the role of the Connection_zone for the Interconnect_component_join_terminal. The connection is optional and need not be supplied if the source system does not contain the data. If the source system contains the data, then the data shall be provided.

4.2.439.2 name

The name specifies the string for the human-interpretable identifier for this Interconnect_component_join_terminal.

Formal constraints:

UR1: The combination of the name and the inherited associated_component shall be unique within a population of Interconnect_component_join_terminal.

WR1: The shapes that describe the connection_zone specified by connections shall reside on one or more surfaces of the assembly_component in the role of associated_component specified by the supertype Component_feature of this terminal.

WR2: Interconnect_component_join_terminal shall participate in no more than one Assembly_joint.
