4.2.257 Delete_design_object_management_relationship

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Delete_design_object_management_relationship is a type of Design_object_management_relationship. Together with Change_design_object_management_relationship and Add_design_object_management_relationship, Delete_design_object_management_relationship supports the exchange of changes to a design by the use of a data set containing only (1) the information necessary to identify the design (e.g., an Ee_product_definition or an Ee_product_version), which is assumed to be already known to the receiving system, and (2) the design objects that have been added, changed, or deleted.

NOTE: Note that all the quality and structural requirements defined in this part of ISO 10303 also apply to the resultant new design iteration.

The data associated with this application object are the following:

  • previous_design_object

4.2.257.1 previous_design_object

The previous_design_object specifies the role of the Managed_design_object for the Delete_design_object_management_relationship.

Formal constraints:

UR1: The combination of the previous_design_object and the previous_design inherited from Design_object_management_relationship shall be unique within a population of Delete_design_object_management_relationships.
