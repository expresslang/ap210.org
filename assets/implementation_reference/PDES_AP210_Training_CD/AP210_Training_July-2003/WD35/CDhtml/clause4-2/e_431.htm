4.2.438 Interconnect_component_interface_terminal

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
An Interconnect_component_interface_terminal is a type of Component_interface_terminal that is some aspect of a generic Assembly_component participate in a connection for a Pca. The Interconnect_component_interface_terminal is defined in-situ. There is no Part_feature associated with it.

EXAMPLES:

  1. An example of an Interconnect_component_interface_terminal is a mechanical conductive fastener that attaches a flexible wire lead lug to the pcb. Current flows through the fastener because it is part of the conduction path for a signal.
  2. A plain wire is considered to be an interconnect component as well. This Interconnect_component_interface_terminal is defined in-situ. There is no Part_feature associated with it. This is the opposite end of the wire from that defined for the same wire in Interconnect_component_join_terminal.

The data associated with this application object are the following:

  • connection
  • name

4.2.438.1 connection

The connection specifies the role of the Connection_zone for the Interconnect_component_interface_terminal. The connection is optional and need not be supplied if the source system does not contain the data. If the source system contains the data, then the data shall be provided.

4.2.438.2 name

The name specifies the string for the human-interpretable identifier for this Interconnect_component_interface_terminal.

Formal constraints:

UR1: The combination of the name and the inherited associated_component shall be unique within a population of Interconnect_component_interface_terminal.

WR1: The shapes that describe the connection_zone specified by connections shall reside on one or more surfaces of the assembly_component in the role of associated_component specified by the supertype component_feature.

WR2: An Interconnect_component_interface_terminal shall participate in only one Assembly_joint.
