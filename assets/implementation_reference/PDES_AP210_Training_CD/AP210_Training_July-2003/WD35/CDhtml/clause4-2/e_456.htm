4.2.463 Interconnect_module_terminal

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
An Interconnect_module_terminal is a type of Part_terminal. The Interconnect_module_terminal shall be one of either an Interconnect_module_interface_terminal or an Interconnect_module_join_terminal. The Interconnect_module_terminal identification is provided by the name attribute inherited from Shape_element. The string may include a substring that indicates to a person the connector the Interconnect_module_terminal is associated with. An Interconnect_module_terminal may be referenced by a member of Part_connected_terminals_definition to identify a connection requirement for the Interconnect_module.

NOTE: The Interconnect_module_terminal may be an Interconnect_module_stratum_based_terminal but this fact is not structurally captured in this application object due to the need to require that the Interconnect_module_terminal shall be one of the two specified subtypes.

NOTE: This part of ISO 10303 does not explicitly model, in a usage view, the relationship of a printed connector to an Interconnect_module in addition to the associated_definition relationship provided in this Application Object.

EXAMPLE: Figure imt illustrates a detail of a usage view that shows an Interconnect_module_terminal that is a part of a edge finger connector.

The data associated with this application object are the following:

  • associated_definition
  • feature_shape
  • terminal_connection_zone

4.2.463.1 associated_definition

The associated_definition specifies the role of the Interconnect_module_usage_view for the Interconnect_module_terminal.

4.2.463.2 feature_shape

The feature_shape specifies the role of the Feature_shape_occurrence for the Interconnect_module_terminal. There shall be one or more Feature_shape_occurrence for an Interconnect_module_terminal.

4.2.463.3 terminal_connection_zone

The terminal_connection_zone specifies the role of the Connection_zone for the Interconnect_module_terminal. There may be more than one Connection_zone for an Interconnect_module_terminal. The terminal_connection_zone is optional and need not be supplied if the source system does not contain the data. If the source system contains the data, then the data shall be provided.

NOTE: The Feature_shape_occurrence specified by the terminal or mounting feature may be the same Feature_shape_occurrence that is specified by the associated Connection_zone. In this case the whole terminal is the Connection_zone.

ENTITY interconnect_module_terminal
  SUPERTYPE OF (ONEOF(
       interconnect_module_interface_terminal,
       interconnect_module_join_terminal))
  SUBTYPE OF (part_terminal);
    terminal_connection_zone                : OPTIONAL SET [1:?] OF connection_zone;
    SELF\part_feature.associated_definition : interconnect_module_usage_view;
INVERSE
    SELF\part_feature.feature_shape         : SET [1:?] OF usage_concept_usage_relationship FOR
    associated_usage;
END_ENTITY; -- interconnect_module_terminal
