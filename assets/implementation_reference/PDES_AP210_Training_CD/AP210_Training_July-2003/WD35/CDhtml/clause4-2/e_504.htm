4.2.504 Layout_macro_component

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Layout_macro_component is a type of Printed_component. A Layout_macro_component uses a Layout_macro_floor_plan_template as the usage definition for the shape of the unexpanded macro in the design. The design definition is the contents of the macro to be expanded. A Layout_macro_component may be exchanged as part of a recursive layout definition. A Layout_macro_component itself generates no Stratum_features since it is not expanded. A Layout_macro_component does inherit terminals from the Printed_component. These terminals are used in the floor planning layout process. These terminals may be on any layer necessary in the design.

NOTE: The recursive definition must end on a Laminate_component that is not a Layout_macro_component.

The data associated with this application object are the following:

  • design_definition
  • usage_definition

4.2.504.1 design_definition

The design_definition specifies the role of a Layout_macro_definitionfor the Layout_macro_component.

4.2.504.2 usage_definition

The usage_definition specifies the role of a Layout_macro_floor_plan_templatefor the Layout_macro_component.

ENTITY layout_macro_component
  SUBTYPE OF (printed_component);
    design_definition : layout_macro_definition;
    SELF\printed_component.usage_definition : layout_macro_floor_plan_template;
WHERE
WR1: NOT(EXISTS (SELF\printed_component.stratum_feature_implementation))
     AND
     NOT(EXISTS (SELF\printed_component.printed_component_stack))
     AND
     NOT(EXISTS (SELF\printed_component.associated_precedent_link))
     AND
     NOT(EXISTS (SELF\printed_component.associated_subsequent_link))
     AND
     NOT(EXISTS (SELF\printed_component.required_material_stack));
END_ENTITY; -- layout_macro_component
