4.2.323 Ee_product_definition

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
An Ee_product_definition is a type of Managed_design_object. An Ee_product_definition is either a Functional_unit_definition or a Physical_unit or additionally may be an Externally_defined_ee_product_definition or additionally may be a Library_ee_product_definition.

NOTE: This entity may be used to capture the definition of a particular Ee_product_version at any intermediate stage of its development where the definition is not formally tracked by an organization at the Ee_product_version level. It may be used to capture the various stages in the definition cycle of a product.

The data associated with this application object are the following:

  • cad_filename
  • context_description
  • creation_date
  • creator
  • discipline_id
  • documentation
  • product_definition_approval
  • product_definition_id
  • security_code
  • version

4.2.323.1 cad_filename

The cad_filename specifies the name of the file or data repository that contains the detailed Ee_product_definition in a computer aided design system. The reference is interpreted to be to an external definition of the details of the Ee_product_definition that is meaningful only within the organization that designed the product. The cad_filename need not be specified for a particular Ee_product_definition.

NOTE: The data contained in the cad_filename may include more information to allow the enterprise to identify the originating system in addition to a simple name.

EXAMPLE: As an example, the following information may be included: operating system supplier name = "xxx", operating system name = "xxx", operating system version = "xxx", cad tool supplier name = "xxx", cad tool name = "xxx", cad tool version = "xxx", cad filename = "xxx", cad filename date stamp ="xxx" where "xxx" is the value information.

4.2.323.2 context_description

The context_description specifies the Ee_text that is the purpose for a particular definition of the Ee_product_definition. The context_description is optional and need not be supplied.

4.2.323.3 creation_date

The creation_date specifies the Date_and_time at which the Ee_product_definition was first defined.

4.2.323.4 creator

The creator specifies the Person_and_organization or the Organization that initially created the Ee_product_definition. There shall be one or more Person_and_organization or Organization for an Ee_product_definition.

4.2.323.5 discipline_id

The discipline_id specifies the name of the discipline that creates the Ee_product_definition.

EXAMPLE: Typical usage may include one of the following as the last item in a comma separated list: analysis, synthesis.

4.2.323.6 documentation

The documentation specifies the role of the Ee_document for the Ee_product_definition. The documentation need not be specified for a particular Ee_product_definition. There may be more than one Ee_document for an Ee_product_definition.

4.2.323.7 product_definition_approval

The product_definition_approval specifies the role of an Ee_approval for the Ee_product_definition.

4.2.323.8 product_definition_id

The product_definition_id specifies the name used for identification of the Ee_product_definition so that users of the information can reference it.

4.2.323.9 security_code

The security_code specifies the role of the Security_classification for the Ee_product_definition. The security_code is optional and need not be supplied for the Ee_product_definition.

4.2.323.10 version

The version specifies the role of the Ee_product_version for the Ee_product_definition.

Formal constraints:

UR1: The combination of the product_definition_id and the version shall be unique within a population of Ee_product_definitions.

UR2: The combination of the discipline_id and the version shall be unique within a population of Ee_product_definitions.

ENTITY ee_product_definition
  ABSTRACT SUPERTYPE OF (ONEOF (
      analytical_model_definition,
      ee_rule_definition,
      evaluation_ee_product_definition,
      footprint_definition,
      functional_unit_definition,
      material_definition,
      padstack_definition,
      physical_unit,
      physical_unit_network_definition,
      requirement_definition,
      stratum,
      template_definition) ANDOR 
      externally_defined_ee_product_definition) 
SUBTYPE OF (managed_design_object);
    creator                     : SET [1:?] OF person_organization;
    product_definition_id       : ee_name;
    security_code               : OPTIONAL security_classification;
    context_description         : OPTIONAL ee_text;
    version                     : ee_product_version;
    product_definition_approval : ee_approval;
    creation_date               : date_and_time;
    documentation               : OPTIONAL SET [1:?] OF ee_document;
    cad_filename                : OPTIONAL ee_name;
    discipline_id               : OPTIONAL ee_text;
    life_cycle_status           : text;
  UNIQUE
    ur1 : product_definition_id, version;
END_ENTITY; -- ee_product_definition
