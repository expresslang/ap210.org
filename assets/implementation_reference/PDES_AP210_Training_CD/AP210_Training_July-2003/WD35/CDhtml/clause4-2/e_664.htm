4.2.664 Physical_unit

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Physical_unit is a type of Ee_product_definition. Each Physical_unit is either a Physical_unit_design_view or a Physical_unit_usage_view. A Physical_unit is the description of physical characteristics common to a variety of realizations of an Ee_product. This description is applicable to every occurrence of a physical realization in composing a higher level assembly.

The data associated with this application object are the following:

  • analytical_physical_unit_representation
  • design_requirement
  • physical_device_characteristic
  • physical_device_coordinated_characteristic
  • unit_technology

4.2.664.1 analytical_physical_unit_representation

The analytical_physical_unit_representation specifies the Analytical_representation for the instance of a computer interpretable model (including assigned parameter values) of the Physical_unit. The analytical_physical_unit_representation need not be specified for a particular subtype of Physical_unit. There may be more than one Analytical_representation for a Physical_unit.

NOTE: Computer interpretable models are used to predict temperatures, stress, and failure rate. This is a representation because all declared parameters have values assigned so that when interpreted by a computer a report may be generated that describes the behaviour of the item represented. This report may be a graph or table.

4.2.664.2 design_requirement

The design_requirement specifies the Ee_requirement_occurrence for the technical requirement for the Physical_unit. The design_requirement need not be specified for a particular Physical_unit. There may be more than one Ee_requirement_occurrence for a Physical_unit.

4.2.664.3 physical_device_characteristic

The physical_device_characteristic specifies the role of the Characteristic for the Physical_unit. The physical_device_characteristic need not be specified for a particular Physical_unit. There may be more than one Characteristic for a Physical_unit.

4.2.664.4 physical_device_coordinated_characteristic

The physical_device_coordinated_characteristic specifies the role of the Coordinated_characteristic for the Physical_unit. The physical_device_coordinated_characteristic need not be specified for a particular Physical_unit. There may be more than one Coordinated_characteristic for a Physical_unit.

4.2.664.5 unit_technology

The unit_technology specifies the Technology used to realize the Physical_unit. The unit_technology need not be specified for a particular Physical_unit.

EXAMPLES:

  1. Examples of the unit_technologies for the Physical_units that are Physical_unit_design_views called Interconnect_modules include rigid, flexible, and flex-rigid multilayer printed circuit boards, and three-dimensional Molded Interconnect Devices. Another categorization of such Physical_units includes digital, analogue, mixed, and rf technologies.
  2. Some examples of the unit_technologies for Physical_units that are Physical_unit_usage_views called Packages include ceramic Dual Inline (DIP), plastic DIP, Single Inline (SIP), J-Lead, metal can, ball grid array, molded axial, and molded radial packages.
  3. Examples of the unit_technologies for the Physical_units that are Physical_unit_usage_views called Bare_die include CMOS, ECL, GAAS, BICMOS.
  4. Examples of the unit_technologies for the Physical_units that are Physical_unit_design_views called Assembly_modules are MCM-D (Multichip Modules, Deposited), MCM-L (Multichip Modules, Laminated), MCM-C (Multichip Modules, Ceramic), and hybrid microcircuits.

NOTE: For interoperability purposes, the technology data also may be included in a more generic manner in Ee_product_category.

ENTITY physical_unit
  ABSTRACT SUPERTYPE OF (ONEOF (physical_unit_design_view,
      physical_unit_usage_view))
  SUBTYPE OF (ee_product_definition);
    analytical_physical_unit_representation    : OPTIONAL SET [1:?] OF 
                                                 analytical_representation;
    unit_technology                            : OPTIONAL technology;
    physical_device_characteristic             : OPTIONAL SET [1:?] OF
                                                 characteristic;
    physical_device_coordinated_characteristic : OPTIONAL SET [1:?] OF 
                                                 coordinated_characteristic;
    design_requirement                         : OPTIONAL SET [1:?] OF 
                                                 ee_requirement_occurrence;
END_ENTITY; -- physical_unit
