4.2.32 Assembly_alternate_product

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
An Assembly_alternate_product is a type of Alternate_select_product where the alternate is valid only for a particular design. The alternate is valid for all instances of the defining_base_product inherited from Alternate_select_product within that design.

NOTE: In specifying the components to be used in an assembly, it is a fairly common practice to stipulate that any usage of a given component (as referenced by a part number, for example) in that assembly may be replaced by some other specified component. Each of those other components is then an Assembly_alternate_product for the specified component (defining_base_product) for that assembly. The Assembly_alternate_product supports only this assembly-level alternate: i.e., it only refers to components that can serve as alternates for any usage of the specified component (the defining_base_product) in that assembly.

The data associated with this application object are the following:

  • basis_context

4.2.32.1 basis_context

The basis_context specifies the role of the Physical_unit_design_view for the Assembly_alternate_product.
