4.2.50 Assembly_module_terminal

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
An Assembly_module_terminal is a type of Part_terminal. An Assembly_module_terminal may be a Pca_terminal. An Assembly_module_terminal may be associated with an instance of Assembly_module_usage_view_connector_relationship to specify connector level interface definition for an assembly. This association is maintained in the usage view. If available, the terminal_connection_zone shall be supplied.

NOTE: The Assembly_module_terminal supports multi-level design, analysis, and verification in a multi-discipline environment. Explicit description of assembly_module_terminals allows connectivity requirements to be explicitly stated and interference requirements to be identified early in the design and development cycle.

The data associated with this application object are the following:

  • associated_definition
  • feature_shape
  • reference_terminal
  • related_connector
  • terminal_connection_zone

4.2.50.1 associated_definition

The associated_definition specifies the role of the Assembly_module_usage_view for the Assembly_module_terminal.

4.2.50.2 feature_shape

The feature_shape specifies the role of the Feature_shape_occurrence for the Assembly_module_terminal. There shall be one or more Feature_shape_occurrence for an Assembly_module_terminal.

4.2.50.3 reference_terminal

The reference_terminal specifies a Packaged_part_interface_terminal for the Assembly_module_terminal. A reference_terminal need not be specified for a Package_part_interface_terminal.

4.2.50.4 related_connector

The related_connector specifies an Assembly_module_usage_view_connector_relationship for the Assembly_module_terminal. A related_connector need not be specified for a Package_part_interface_terminal.

4.2.50.5 terminal_connection_zone

The terminal_connection_zone specifies the role of the Connection_zone for the Assembly_module_terminal. There may be more than one Connection_zone for an Assembly_module_terminal. The terminal_connection_zone is optional and need not be supplied. If the data is available in the source system, it shall be supplied.

NOTE: The Feature_shape_occurrence specified by the terminal or mounting feature may be the same Feature_shape_occurrence that is specified by the associated Connection_zone. In this case the whole terminal is the Connection_zone.

Informal constraints:

IP1: If a related_connector is specified, the position,shape, and orientation of the Assembly_module_terminal shall be overriden by the reference_terminal position, shape, and orientation.

ENTITY assembly_module_terminal
  SUPERTYPE OF ((ONEOF(
      assembly_module_interface_terminal, 
      assembly_module_join_terminal)) 
     ANDOR (pca_terminal))
  SUBTYPE OF (part_terminal);
    SELF\part_feature.associated_definition : assembly_module_usage_view;
    reference_terminal                      : 
	OPTIONAL packaged_part_interface_terminal; 
    related_connector                       : 
	OPTIONAL assembly_module_usage_view_connector_relationship;
    terminal_connection_zone                : OPTIONAL SET [1:?] OF connection_zone;
INVERSE
    SELF\part_feature.feature_shape         : SET [1:?] OF usage_concept_usage_relationship FOR
    associated_usage;
END_ENTITY; -- assembly_module_terminal
