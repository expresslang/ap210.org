4.2.177 Configuration_controlled_message

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Configuration_controlled_message is a type of Managed_design_object. A Configuration_controlled_message is a message that has its content under version control, and that is source data for application of marking materials to a product.

The data associated with this application object are the following:

  • message_content
  • message_source
  • message_type_specification

4.2.177.1 message_content

The message_content specifies the text in the Configuration_controlled_message.

4.2.177.2 message_source

The message_source specifies either an Ee_approval, an Ee_product_version, or an Organization for the Configuration_controlled_message.

4.2.177.3 message_type_specification

The message_type_specification specifies either an approval_date, a manufacturer_code, a part_number, or a revision_identifier for the Configuration_controlled_message. A message_type_specification is necessary to differentiate among the possible message sources to identify the one that has been selected for the literal string.

Formal constraints:

WR1: If message_type_specification is "approval_date" then message_content is equal to Ee_approval.signoff_date.

WR2: If message_type_specification is "manufacturer_code" then message_content is equal to Organization.identifier.

WR3: If message_type_specification is "part_number" then message_content is equal to Ee_product.part_number.

WR4: If message_type_specification is "revision_code" then message_content is equal to Ee_product_version.revision_code.
