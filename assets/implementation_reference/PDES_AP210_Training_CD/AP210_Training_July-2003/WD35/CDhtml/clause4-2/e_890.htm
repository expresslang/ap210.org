4.2.890 Terminal_pair_port_equivalence_relationship

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Terminal_pair_port_equivalence_relationship is a type of Managed_design_object. A Terminal_pair_port_equivalence_relationship is the assignment association between a pair of scalar ports and a single distributed port.

The data associated with this application object are the following:

  • equivalent_port
  • scalar_ports

4.2.890.1 equivalent_port

The equivalent_port specifies the Analytical_model_port that represents the single distributed port for the Terminal_pair_port_equivalence_relationship.

4.2.890.2 scalar_ports

The scalar_ports specify the two Analytical_model_scalar_port in the ordered sequence for the Terminal_pair_port_equivalence_relationship. The second scalar_port is always the reference port. Other details depend on the specific domain of the Analytical_model.

EXAMPLE: Port "INPUT" of a microwave coaxial filter model is the equivalent_port for this example. Port "INPUT center pin" is the first Analytical_model_scalar_port for this example. Port "INPUT shield" is the second Analytical_model_scalar_port for this example.

Formal constraints:

WR1: The distributed_properties attribute of the Analytical_model_port in the role of equivalent_port shall be "TRUE".

WR2: The scalar_ports and equivalent_port shall not reference the same Analytical_model.

WR3: The scalar_ports shall reference the same Analytical_model.

WR4: The equivalent_port shall not be a member of Digital_analytical_model_port.

ENTITY terminal_pair_port_equivalence_relationship
  SUBTYPE OF (managed_design_object);
    equivalent_port : analytical_model_port;
    scalar_ports    : LIST [2:2] OF UNIQUE analytical_model_scalar_port;
END_ENTITY; -- terminal_pair_port_equivalence_relationship
