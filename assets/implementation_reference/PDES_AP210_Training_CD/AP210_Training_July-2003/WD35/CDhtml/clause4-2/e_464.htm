4.2.470 Interface_mounted_join

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
An Interface_mounted_join is a type of Managed_design_object. An Interface_mounted_join is the relationship that specifies the connectivity between a terminal on a component and the terminal on a socket in the assembly that directly mates with that component terminal.

EXAMPLE: An example of this relation is pin 1, a Component_terminal, on U1, an integrated circuit and a component, that is connected to pin 1, on XU1, a socket and an Interface_component, when the IC is inserted into the socket in the assembly.

NOTE: This Application object is not referencing the terminal on the socket that mates with the base interconnect substrate. That is referenced by a member of Assembly_joint.

The data associated with this application object are the following:

  • access_to_interface_mounted_component
  • connection_to_interconnect_module_component

4.2.470.1 access_to_interface_mounted_component

The access_to_interface_mounted_component specifies the Component_terminal of the Assembly_component being interfaced to the assembly by the Interface_component specified by connection_to_interconnect_module_component.

4.2.470.2 connection_to_interconnect_module_component

The connection_to_interconnect_module_component specifies the Component_interface_terminal that mates with the Component_terminal specified by access_to_interface_mounted_component. The Component_interface_terminal is a terminal of the Interface_component.

Formal constraints:

WR1: A Physical_connectivity_definition shall not specify a Component_terminal that participates in an Interface_mounted_join relationship as access_to_interface_mounted_component if that Physical_connectivity_definition specifies the Component_terminal of the associated Interface_component that completes connectivity to the board from the interface_mounted_component.

WR2: All members of Component_terminal specified by associated_terminals shall be in the same design view.

NOTE: It is inappropriate to specify both the terminal on an IC plugged into a socket and the socket terminal in the same nodelist. It is unnecessary to specify the interface terminal on a socket in a nodelist, as the socket terminal is referenced by a member of Interface_mounted_join that is referenced by a member of Physical_node_requirement_to_implementing_component_allocation that references the original nodelist.

WR3: Both components whose terminals are being joined shall be for the same version of the Product.
