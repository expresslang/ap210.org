4.2.391 Footprint_occurrence_sub_assembly_relationship

A Footprint_occurrence_sub_assembly_relationship is a type of Managed_design_object. A Footprint_occurrence_sub_assembly_relationship is an association that specifies a subassembly constraint among two or more Land that is aggregated into an Footprint_occurrence. A Footprint_occurrence_sub_assembly_relationship does not directly reference the Application Object that is the definition for the Land. Instead, it uses an indexing mechanism provided by terminal_location and sub_assembly_refence_designation to locate the padstack in the Footprint_definition and the land template location in that Padstack_definition.

EXAMPLE: A Land may participate in a Footprint_occurrence. In that case, there is no need to populate a separate Assembly_composition_relationship between the Land and the Pcb, since the Assembly_composition_relationship between the Footprint_occurrence and the Pcb establishes the required association.

The data associated with this application object are the following:

  • related_component
  • relating_component

4.2.391.1 terminal_location

The terminal_location specifies the string which is the identifier for the Padstack_location_in_footprint which this Footprint_occurrence_sub_assembly_relationship references in the context of the definition of the relating_component.

4.2.391.2 sub_assembly_reference_designation

The sub_assembly_reference_designation specifies the string which is the identifier for the Land in the context of the Padstack_definition specified by the definition of the relating_component.

4.2.391.3 related_component

The related_component specifies the dependent Land for the Footprint_occurrence_sub_assembly_relationship.

4.2.391.4 relating_component

The relating_component specifies the independent Footprint_occurrence which is the aggregation of two or more Assembly_component specified by related_component.

Formal constraints:

UR1: The combination of relating_component, terminal_location, and sub_assembly_reference_designation shall be unique in a population of Footprint_occurrence_sub_assembly_relationship.

ENTITY footprint_occurrence_sub_assembly_relationship
  SUBTYPE OF (managed_design_object);
    related_component : laminate_component;
    relating_component : footprint_occurrence;
    sub_assembly_reference_designation : string;
    terminal_location : string;
UNIQUE
  UR1: sub_assembly_reference_designation, 
       relating_component, 
       terminal_location;
END_ENTITY; -- footprint_occurrence_sub_assembly_relationship
