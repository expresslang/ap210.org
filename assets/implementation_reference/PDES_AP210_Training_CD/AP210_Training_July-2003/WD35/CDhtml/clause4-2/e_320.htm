4.2.320 Ee_product_category

An Ee_product_category is a type of Managed_design_object. An Ee_product_category is a class in a class hierarchy for the purpose of categorizing products according to their characteristics. An Ee_product_category may be an Ee_product_related_product_category.

The data associated with this application object are the following:

  • category_parameter
  • category_specific_values
  • description
  • name
  • reference_document

4.2.320.1 category_parameter

The category_parameter specifies the Model_parameter assigned to this Ee_product_category for categorization purposes. All sub-categories of this Ee_product_category shall inherit the category_parameter. There shall be one or more Model_parameter for an Ee_product_category.

4.2.320.2 category_specific_values

The category_specific_values specifies the Parameter_assignment that assigns a specific value to a specific Model_parameter for this Ee_product_category. All sub-categories of this Ee_product_category shall inherit the category_specific_values. There shall be one or more Parameter_assignment for an Ee_product_category.

4.2.320.3 description

The description specifies one role of a string for the Ee_product_category.

4.2.320.4 name

The name specifies the string for the identifier for the Ee_product_category.

4.2.320.5 reference_document

The reference_document specifies the Ee_document that is the reference for the Ee_product_category. The reference_document establishes the scope for the name. All Ee_product_category names defined in this standard shall use this standard as the reference_document.

NOTE: The use of reference_document allows organizations to resolve name clashes between Ee_product_category instances.

EXAMPLE: This standard pre-defines the name 'assembly' while an organization might use 'PWA' for that purpose. The inclusion of the reference_document permits both to be in a product category tree and pointing to the same products. An application conformant to this standard looking for pre-defined names would use the reference_document as a filter and ignore 'PWA'.

Formal constraints:

UR1: The name in combination with the reference_document shall be unique within a population of Ee_product_categories within an enterprise.

WR1: Each category_specific_value shall be assigned to one member of Model_parameter that is in the role of category_parameter in a higher level category of this Ee_product_category.

WR2: All Ee_product_category names defined in this standard shall use this standard as the reference_document.

ENTITY ee_product_category
  SUPERTYPE OF (ee_product_related_product_category)
  SUBTYPE OF (managed_design_object);
    category_parameter       : OPTIONAL SET [1:?] OF model_parameter;
    category_specific_values : OPTIONAL SET [1:?] OF parameter_assignment;
    description              : OPTIONAL ee_name;
    name                     : ee_name;
    reference_document       : ee_document;
UNIQUE
  ur1: name, reference_document;
END_ENTITY; -- ee_product_category
