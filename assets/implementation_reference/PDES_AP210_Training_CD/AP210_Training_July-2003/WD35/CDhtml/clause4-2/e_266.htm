4.2.266 Design_functional_unit_allocation_to_reference_functional_unit

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Design_functional_unit_allocation_to_reference_functional_unit is a type of Managed_design_object. A Design_functional_unit_allocation_to_reference_functional_unit is an assignment of a Functional_unit in a Design_composition_path to a Functional_unit in a Reference_composition_path for the functional definition of a product.

NOTES:

  1. This provides the capability to exchange a functional decomposition that an enterprise reuses in multiple designs.
  2. The library functional hierarchy, if provided, is required to be a network definition with the constraint that only external references on the root node of the network structure will be mapped to physical terminals of the reused product design.
  3. The allocation relationship in this Application Object is critical in mapping between a functional design and a physical implementation of that design, because this part of ISO 10303 relies on the reference mapping between a functional definition in a library and the physical implementation of that functional definition in that library, usually in a packaged part, to identify the necessary definitions that need to be instantiated in the physical design.

The data associated with this application object are the following:

  • design_definition_path
  • reference_definition_path

4.2.266.1 design_definition_path

The design_definition_path specifies the role of the Design_composition_path for the Design_functional_unit_allocation_to_reference_functional_unit.

4.2.266.2 reference_definition_path

The reference_definition_path specifies the role of the Reference_composition_path for the Design_functional_unit_allocation_to_reference_functional_unit.

Formal constraints:

WR1: The functional_unit_definition that defines the leaf node of the reference_definition_path and the functional_unit_definition that defines the leaf node of the design_definition_path shall be the same definition.
