4.2.660 Physical_network

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Physical_network is a type of Managed_design_object. A Physical_network may be a Partially_routed_physical_network. A Physical_network is the result of mapping a topological definition of a physical connectivity requirement onto the conducting Design_layer_stratum of an Interconnect_module. The vertices, specified by Component_terminal, are mapped onto Layer_connection_point on the surface of one or more conducting Design_layer_stratum in the Interconnect_module. The design features that are implementing the Physical_network are accessed by traversing the structure specified by the interconnect_module_net_topology_element attribute. The Physical_network shall not implement direct connections to Component_terminal specified by Interface_mounted_join, or Assembly_joint, but shall implement connections only to Component_terminals in the scope of the design view being implemented. This Physical_network implements connectivity to the join terminals of the components that are interface components in this Interconnect_module, which is a design view. The scope of the Physical_network is entirely within the interconnect design view and provides no information about assembly connectivity of design views within which occurrences of the Interconnect_module, as Interconnect_module_component may appear. The Physical_network shall not use as connectivity_requirement_element terminals of Land or Non_functional_land that are supporting internal implementation of the Physical_network. Only Land or Non_functional_land types that have interface_terminals shall be referenced.

NOTE: The Physical_node_requirement_to_implementing_component_allocation application object is provided so that traceability of the required connectivity for an assembly design view to that implemented internally in an Interconnect_module may be supported. The Part_connected_terminals_definition application object is provided to specify explicitly the external terminals of the interconnect substrate that are to be connected. In a case where a design organization re-uses an interconnect design, the Part_connected_terminals_definition, since it is the actual connection statement for an Interconnect_module, defines the capabilities of the interconnect substrate to satisfy the requirements imposed by the Physical_node_requirement_to_implementing_component_allocation object. There may be many sets of Physical_node_requirement_to_implementing_component_allocation (one set per assembly design) that reference an interconnect substrate. There is only one set of Part_connected_terminals_definition for an interconnect substrate.

The data associated with this application object are the following:

  • current_associated_definition
  • prior_associated_definition
  • connectivity_requirement_element
  • interconnect_module_net_topology_element
  • name

4.2.660.1 current_associated_definition

The current_associated_definition specifies the Interconnect_module that is the current product definition associated with the Physical_network.

4.2.660.2 prior_associated_definition

The prior_associated_definition specifies one or more Interconnect_module that are definitions for prior versions of the design. The prior_associated_definition need not be provided.

4.2.660.3 connectivity_requirement_element

The connectivity_requirement_element specifies the Component_terminal to be connected by the network described by the Physical_network. There shall be two or more Component_terminal for a Physical_network.

4.2.660.4 interconnect_module_net_topology_element

The interconnect_module_net_topology_element specifies the unique Join_relationship that results from mapping the topological statement of the physical connectivity requirement onto the coordinate space of the conducting Design_layer_stratum that compose the Interconnect_module. There shall be one or more Join_relationship for a Physical_network.

4.2.660.5 name

The name identifies the Physical_network.

Formal constraints:

UR1: The name shall be unique within a population of Physical_networks in the context of the current_associated_definition.

WR1: The Component_terminal specified by connectivity_requirement_element shall be one of the following: Land_join_terminal, Non_functional_land_join_terminal, Printed_component_join_terminal, Via_terminal, Component_termination_passage_join_terminal.

WR2: All members of connectivity_requirement_element and interconnect_module_network_topology_element shall be in the current_associated_definition.

WR3: If the prior_associated_definition exists, then all members of connectivity_requirement_element and interconnect_module_network_topology_element shall be in the prior_associated_definition.

WR4: If the prior_associated_definition exists, then each member of prior_associated_definition shall not be an current_associated_definition.

WR5: Each Ee_material that participates in the Physical_network shall be the same material.

WR6: Each member of Component_terminal in the role of connectivity_requirement_element and each member of Join_relationship in the role of interconnect_module_network_topology_element referenced by Physical_network shall be in the same design view or shall be an explicit external reference.

NOTE: A Physical_network does not include connection requirements that are implemented by a Printed_component.

ENTITY physical_network
  SUPERTYPE OF (partially_routed_physical_network)
  SUBTYPE OF (managed_design_object);
    current_associated_definition                : interconnect_module;
    prior_associated_definition                  : OPTIONAL SET [1:?] OF interconnect_module;
    connectivity_requirement_element             : SET [2:?] OF laminate_component_join_terminal;
    interconnect_module_network_topology_element : SET [1:?] OF join_relationship;
    name                                         : ee_name;
    reference_connected_terminals                : OPTIONAL part_connected_terminals_definition;
  UNIQUE
    ur1 : name, current_associated_definition;
WHERE
  WR1: NOT(EXISTS(prior_associated_definition)) OR
    NOT(current_associated_definition IN prior_associated_definition);
  WR2: SIZEOF(QUERY(cre <* connectivity_requirement_element |
       NOT(NOT('AP210_ARM.LAND_JOIN_TERMINAL' IN TYPEOF(cre))
           OR (SIZEOF(TYPEOF(cre\component_feature.associated_component) *
          ['AP210_ARM.CONTACT_SIZE_DEPENDENT_LAND',
           'AP210_ARM.' +
       'COMPONENT_TERMINATION_PASSAGE_AND_CONTACT_SIZE_DEPENDENT_LAND',
           'AP210_ARM.VIA_AND_CONTACT_SIZE_DEPENDENT_LAND']) = 1))
          )) = 0;
  WR3: SIZEOF(QUERY(cre <* connectivity_requirement_element | 
       NOT(NOT('AP210_ARM.NON_FUNCTIONAL_LAND_JOIN_TERMINAL' IN TYPEOF(cre))
           OR (SIZEOF(TYPEOF(cre\component_feature.associated_component) *
          ['AP210_ARM.CONTACT_SIZE_DEPENDENT_NON_FUNCTIONAL_LAND',
           'AP210_ARM.' +
  'COMPONENT_TERMINATION_PASSAGE_AND_CONTACT_SIZE_DEPENDENT_NON_FUNCTIONAL_LAND',
           'AP210_ARM.VIA_AND_CONTACT_SIZE_DEPENDENT_LAND']) = 1))
          )) = 0;

END_ENTITY; -- physical_network
