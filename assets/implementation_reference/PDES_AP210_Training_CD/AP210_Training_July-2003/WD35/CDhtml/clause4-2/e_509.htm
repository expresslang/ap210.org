4.2.509 Layout_spacing_requirement_occurrence

NOTE: The text description for this Application Object has not been fully updated to agree with the EXPRESS.
A Layout_spacing_requirement_occurrence is a type of Ee_requirement_occurrence that describes the physical spacing between one of an assembly_module_assembly_component, an assembly_module_component_feature, an interconnect_module_cutout, an interconnect_module_fill_area, an interconnect_module_design_intent_modification, an interconnect_module_inter_stratum_feature, an interconnect_module_cutout_edge_segment, an interconnect_module_interconnect_module_edge, an interconnect_module_interconnect_module_edge_segment, an interconnect_module_plated_cutout, an interconnect_module_plated_cutout_edge_segment, an interconnect_module_plated_interconnect_module_edge, an interconnect_module_plated_interconnect_module_edge_segment, an interconnect_module_plated_passage, a category consisting of the union of an interconnect_module_plated_passage category and an interconnect_module_plated_unsupported_passage category, an interconnect_module_unsupported_passage, an interconnect_module_stratum_feature, or an interconnect_module_via, and another of these categories of design objects. A Layout_spacing_requirement_occurrence may be a Layer_qualified_layout_spacing_requirement_occurrence. The requirement shall be imposed between the boundaries of members of the the relevant categories or between centroids. The category names correspond to the category of equivalent application object names.

NOTE: Support for requirements between boundaries and centroids is not supported by this part of ISO 10303.

EXAMPLE: Assembly_module_assembly_component corresponds to Assembly_component category. Assembly_module_component_feature corresponds to Component_feature category. Interconnect_module_cutout corresponds to Cutout category. Interconnect_module_fill_area corresponds to Fill_area category. Interconnect_module_inter_stratum_feature corresponds to Inter_stratum_feature category. Interconnect_module_stratum_feature corresponds to Stratum_feature category. Interconnect_module_via corresponds to Via category.
Any subtype of an item in a category shall qualify as part of that category.

NOTES:

  1. This part of ISO 10303 supports several spacing requirements in this Application Object.
  2. A pair of categories is used for this requirement occurrence.

EXAMPLES:

  1. In domain terms, when a Via to Via spacing is to be greater than 5 mm, the encoding will be that the spacing between the reference_design_object_category whose value is interconnect_module_via and the dependent_design_object_category whose value is interconnect_module_via shall be greater than 5 mm.
  2. In domain terms, when the distance from a cutout to a trace must be greater than 5 mm, the encoding will be that the spacing between the reference_design_object_category whose value is interconnect_module_cutout and the dependent_design_object_category whose value is interconnect_module_stratum_feature shall be greater than 5 mm.

The data associated with this application object are the following:

  • dependent_design_object_category
  • required_coordinated_characteristic
  • reference_design_object_category
  • of_spacing_type

4.2.509.1 dependent_design_object_category

The dependent_design_object_category specifies one role of either an assembly_module_assembly_component, an assembly_module_component_feature, an interconnect_module_cutout, an interconnect_module_fill_area, an interconnect_module_inter_stratum_feature, an interconnect_module_stratum_feature, or an interconnect_module_via, for the Layout_spacing_requirement_occurrence.

4.2.509.2 required_coordinated_characteristic

The required_coordinated_characteristic inherited from the Ee_requirement_occurrence supertype specifies the Tolerance_characteristic for the Layout_spacing_requirement_occurrence.

4.2.509.3 reference_design_object_category

The reference_design_object_category specifies one role of either an assembly_module_assembly_component, an assembly_module_component_feature, an interconnect_module_cutout, an interconnect_module_fill_area, an interconnect_module_inter_stratum_feature, an interconnect_module_stratum_feature, or an interconnect_module_via, for the Layout_spacing_requirement_occurrence.

4.2.509.4 of_spacing_type

The of_spacing_type is either "centroid", or "nearest_boundary", or "furthest_boundary" for the Layout_spacing_requirement_occurrence.

Formal constraints:

UR1: The combination of the reference_design_object_category and the dependent_design_object_category shall be unique within a population of Layout_spacing_requirement_occurrences.

ENTITY layout_spacing_requirement_occurrence
  SUPERTYPE OF (layer_qualified_layout_spacing_requirement_occurrence)
  SUBTYPE OF (ee_requirement_occurrence);
    reference_design_object_category : pca_product_design_object_category;
    dependent_design_object_category : pca_product_design_object_category;
   SELF\ee_requirement_occurrence.required_coordinated_characteristic :
                                         tolerance_characteristic;
    of_spacing_type                     : spacing_type;
UNIQUE
  ur1: reference_design_object_category, dependent_design_object_category;
WHERE
  wr1: 'AP210_ARM.LENGTH_DATA_ELEMENT' IN
          TYPEOF(SELF\ee_requirement_occurrence.
           required_coordinated_characteristic[1].component_characteristic[1]);
END_ENTITY; -- layout_spacing_requirement_occurrence
